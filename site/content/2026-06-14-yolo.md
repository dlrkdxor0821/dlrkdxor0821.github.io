# YOLO 학습 노트

객체 탐지 모델 YOLO 개념 정리. 학습/저장/배포 흐름, 모델 종류, 동작 원리, 학습·추론 옵션, 결과 그래프 해석, 핵심 요약까지.

---

## 1. YOLO란? — "학습시킨다"의 의미

YOLO(You Only Look Once)는 이미지를 보고 한 번에 다음을 예측하는 **객체 탐지(Object Detection)** 모델이다.

1. **어디에** 물체가 있는지 — Bounding Box (박스 좌표)
2. **무엇인지** — 클래스 (사람/자동차/책 …)
3. **얼마나 확신하는지** — Confidence

**"학습시킨다(train)"** = 모델 내부의 수백만 개 가중치(weight)를 데이터로부터 조정해, 예측이 정답에 가까워지도록 최적화하는 과정.

```
1. 이미지 입력  →  2. 모델이 박스/클래스 예측  →  3. 정답(라벨)과 비교
                                                        ↓
6. 가중치 업데이트  ←  5. 오차를 거꾸로 전파  ←  4. 오차(Loss) 계산
        ↑                                              
        └──────────── 이걸 수천~수만 번 반복 ────────────┘
```

**학습에 필요한 것**: 데이터셋(이미지 + 라벨), 라벨링(정답 박스/클래스 표시), GPU, 하이퍼파라미터(epoch·learning rate·batch 등).

> 비유: 데이터셋=문제집+정답지, 학습=풀고 채점하고 틀린 것 고치기 반복, 학습된 모델=새 문제도 풀 수 있는 상태.

## 2. YOLO 종류 (Task 변형)

같은 YOLO 계열이지만 **푸는 문제(task)** 에 따라 모델이 나뉜다. 가중치 파일명 접미사로 구분된다.

| Task | 작업 | 출력 | 파일 예시 |
|------|------|------|-----------|
| **detect** (기본) | 객체 탐지 | 바운딩 박스 + 클래스 | `yolo11n.pt` |
| **segment** (-seg) | 인스턴스 분할 | 박스 + **픽셀 마스크**(실제 윤곽) | `yolo11n-seg.pt` |
| **pose** | 자세 추정 | 사람 키포인트(관절) | `yolo11n-pose.pt` |
| **classify** (-cls) | 이미지 분류 | "이 사진이 무슨 종류"(박스 없음) | `yolo11n-cls.pt` |
| **obb** | 회전 박스 탐지 | 기울어진 박스(항공/문서) | `yolo11n-obb.pt` |

### detect vs segment 핵심 차이

| | detect | segment (-seg) |
|---|---|---|
| 위치 표현 | 네모 박스 | 픽셀 마스크 (실제 모양) |
| 속도 | 더 빠름 | 약간 느림 (마스크 연산 추가) |
| 정밀도 | 대략적 위치 | 정확한 외곽선 |
| 용도 | 개수/위치 파악 | 면적 측정, 배경 제거, 정밀 윤곽 |

> 단순히 "어디 있나/몇 개인가"만 필요 → **detect**.
> "객체의 정확한 모양/면적"까지 필요 → **segment**.

## 3. 학습된 모델은 어디에 저장되나?

학습이 끝나면 모델은 **하나의 파일**로 디스크에 저장된다. Ultralytics YOLO 기준:

```
runs/detect/train/weights/
├── best.pt    ← 학습 중 성능이 가장 좋았던 가중치
└── last.pt    ← 마지막 epoch의 가중치
```

- `.pt` = PyTorch 모델 파일 (가중치 숫자들의 묶음)
- **모델 = 파일 하나(best.pt)** 라고 생각하면 된다. USB로 복사하든 클라우드에 올리든 자유.

### 다른 로컬 환경에서 학습 가능? → 가능
- 내 노트북, 회사 데스크탑, Google Colab(무료 GPU), 클라우드 서버 등 어디서든 가능. 결과물은 똑같은 `.pt` 파일.
- 단, 학습은 연산량이 커서 GPU 없으면 매우 느리다 → GPU 환경 권장.

### HuggingFace에 올리면? → 받아서 쓸 수 있음
HuggingFace = **모델 파일을 올려두고 공유하는 창고**(GitHub의 모델 버전). 올리는 건 저장/공유일 뿐, 거기서 학습이 되는 게 아니다.

```
[학습] best.pt 생성 → [업로드] HuggingFace Hub → [다운로드] 다른 환경에서 그대로 사용
```

| 단계 | 어디서? | 결과물 |
|------|---------|--------|
| 학습(Train) | 아무 환경 — GPU 권장 | `best.pt` 생성 |
| 저장(Save) | 로컬 → 원하면 HuggingFace | 파일 공유 |
| 사용(Inference) | 아무 환경 — CPU도 가능 | 탐지 결과 |

> **그냥 쓰기만** → `.pt` + 라이브러리만 있으면 됨(프로젝트 폴더·데이터셋 불필요).
> **내 데이터로 더 학습(fine-tuning)** → 데이터셋 추가 필요.

```python
from ultralytics import YOLO
model = YOLO("best.pt")          # 파일만 불러오면 끝
results = model("photo.jpg")     # 바로 추론
```

## 4. YOLO 동작 원리

### 4.1 그리드 분할 — 누가 어떤 물체를 맡을지 나누기

입력 이미지를 SxS 그리드로 분할. 규칙: **물체의 "중심"이 들어있는 칸이 그 물체를 예측할 책임**을 진다.

```
┌──┬──┬──┬──┬──┬──┬──┐
│  │  │🐶│  │  │  │  │  ← 강아지 중심이 있는 칸이 강아지 담당
├──┼──┼──┼──┼──┼──┼──┤
│  │  │  │  │  │🚗│  │  ← 자동차 중심이 있는 칸이 자동차 담당
└──┴──┴──┴──┴──┴──┴──┘
```

각 셀은 **Bounding Box(x,y,w,h) + Confidence + Class 확률** 을 예측. 모든 칸이 동시에 예측 → 한 번의 forward pass로 빠르게 처리.

### 4.2 NMS(Non-Maximum Suppression) — 같은 물체의 중복 박스 제거

한 물체에 여러 박스가 겹쳐 잡히면 가장 확실한 하나만 남긴다.

```
[NMS 전] 강아지 1마리에 박스 3개      [NMS 후] 1개만 남김
   ┌────────┐                          ┌────────┐
   │┌───────┴┐                         │  🐶    │
   ││ 🐶     │                          └────────┘
   │└┬───────┘
   └─┴──────┘
```

**"같은 물체"인지 판단 기준 = 겹침 정도(IoU)**
- 많이 겹침(IoU 높음) + 같은 클래스 → 같은 물체 → 점수 낮은 박스 제거
- 안 겹침(IoU 낮음) → 다른 물체 → 둘 다 유지

> **핵심: NMS는 겹침이 큰 박스끼리만 경쟁시켜 하나를 남긴다. 다른 물체(겹침 적음)는 건드리지 않는다.**

### 4.3 1-stage vs 2-stage — 왜 빠른가

```
2-stage (R-CNN 계열): 후보 영역 찾기 → 각각 분류   (정확하지만 느림)
1-stage (YOLO):        한 번에 위치+분류 동시 예측   (빠르고 실시간)
```

YOLO는 탐지를 **단일 신경망의 회귀 문제**로 봐서 영역 제안 단계 없이 한 번에 끝낸다. 전체 이미지를 한 번에 보므로 맥락(context) 파악 → 배경 오탐이 적다. (초기엔 작은·밀집 물체에 약했으나 v3~v11에서 개선)

## 5. 학습 설정 — Epoch / Batch / Iteration

**Epoch(에폭)** = 전체 학습 데이터셋을 처음부터 끝까지 **한 번 다 학습한 횟수**.

> 문제집 한 권 = 전체 데이터셋. 1 epoch = 문제집 1번 다 풀기. 100 epochs = 100번 반복. 여러 번 반복해야 패턴을 제대로 학습한다.

| 용어 | 뜻 |
|------|-----|
| **Epoch** | 전체 데이터를 1번 다 본 것 |
| **Batch** | 한 번에 다 못 보니 잘게 나눈 묶음 (예: 16장씩) |
| **Iteration** | 배치 1개를 학습하는 1번의 단계 |

예) 데이터 1,600장, batch=16 → **1 epoch = 100 iterations** (1600 ÷ 16)

**몇으로 설정?**
- 너무 적으면(예: 5) → 덜 배움 = **과소적합(underfitting)**
- 너무 많으면(예: 1000) → 통째로 외워버려 새 데이터에 약함 = **과적합(overfitting)**
- YOLO 기본값은 보통 `epochs=100`. loss/지표 그래프를 보며 더 이상 안 좋아지면 조절/중단.

```bash
yolo train model=yolo11n.pt data=data.yaml epochs=100 batch=16 imgsz=640
```

## 6. 추론(predict) 옵션 — conf / save

```python
model.predict(source="./sample.jpg", conf=0.2, save=True)
```

### conf=0.2 (confidence threshold, 신뢰도 임계값)
- 예측 확률이 **0.2(20%) 이상인 것만** 결과로 채택, 그 미만은 버림.

| conf 값 | 효과 |
|---------|------|
| 낮게 (0.1) | 애매한 것도 다 잡음 → 탐지↑, 오탐(틀린 것)도↑ |
| 높게 (0.7) | 확실한 것만 잡음 → 정확하지만 놓치는 것 생김 |

- 기본값 0.25. conf는 주로 **detect/segment**에서 효과가 큼. classify(분류) 모델은 박스가 없어 임계값 필터링 의미가 작다.

### save=True (결과 저장 여부)
- 박스/마스크가 그려진 결과 이미지를 파일로 저장.
- `True` → `runs/detect/predict/` 등에 자동 저장. `False`(기본) → 결과 객체만 반환, 파일 안 만듦.

## 7. 학습 결과 그래프 읽기 ★

![YOLO 학습 결과 그래프 — loss 4종과 precision/recall/mAP 지표](https://images.prismic.io/asd0821/ai4Lv41P9HI4UbUK_Screenshotfrom2026-06-1410-41-56.png?auto=format,compress)

학습이 끝나면 Ultralytics가 `results.png`를 생성한다. 가로축 = **epoch**, 파란선 = 실제값(results), 주황 점선 = 추세(smooth). 보통 **윗줄 = train, 아랫줄 = val(검증)**.

### Loss(손실) — 낮을수록 좋음 ↓ (감소해야 정상)

| 지표 | 뜻 |
|------|-----|
| **box_loss** | 박스 위치/크기 오차 (얼마나 정확히 박스를 쳤나) |
| **cls_loss** | 분류 오차 (클래스를 맞게 맞췄나) |
| **dfl_loss** | Distribution Focal Loss — 박스 경계를 더 정밀하게 잡기 위한 오차 |

- `train/*_loss`가 계속 내려가면 학습이 잘 되는 중.
- `val/*_loss`도 같이 내려가야 좋음. **train은 내려가는데 val이 다시 올라가면 → 과적합 신호.**

### Metrics(성능 지표) — 높을수록 좋음 ↑ (증가해야 정상)

| 지표 | 뜻 |
|------|-----|
| **precision(B)** | 정밀도 — 모델이 "있다"한 것 중 진짜 맞은 비율 (오탐 적을수록↑) |
| **recall(B)** | 재현율 — 실제 객체 중 모델이 찾아낸 비율 (놓침 적을수록↑) |
| **mAP50(B)** | IoU 0.5 기준 평균 정밀도. **대표 성능 지표** |
| **mAP50-95(B)** | IoU 0.5~0.95 전 구간 평균. 더 엄격한 종합 지표 |

> `(B)` = **Box** 기준 지표라는 뜻 (segment 모델이면 마스크 기준 `(M)`도 함께 나옴).

### 한 장 요약으로 "잘 됐나" 판단
- ✅ **좋은 학습**: loss 4종이 우하향, precision·recall·mAP 4종이 우상향, train과 val이 비슷하게 따라감.
- ⚠️ **과적합**: train loss는 내려가는데 val loss가 올라감 / val mAP가 정체·하락.
- ⚠️ **학습 부족**: 그래프가 끝까지 계속 좋아지는 중 → epoch를 더 늘릴 여지.

> 위 스크린샷(`results.png`)은 loss 4종이 모두 감소하고 precision/recall/mAP가 모두 상승하는 **전형적인 정상 학습** 패턴이다.

## 8. 핵심 요약

### 30초 버전
> "YOLO는 You Only Look Once의 약자로, 이미지를 한 번만 보고 객체의 위치와 종류를 동시에 예측하는 객체 탐지 모델입니다. 기존 방식이 '후보 영역 찾기 → 분류'를 따로 했다면, YOLO는 이를 하나의 회귀 문제로 통합해 한 번에 처리하므로 매우 빠르고 실시간 처리가 가능합니다."

### 외울 핵심 흐름 5개
**① You Only Look Once → ② 1-stage(한 번에) → ③ 그리드 분할 → ④ 박스+Confidence+Class 동시 예측 → ⑤ NMS로 중복 제거**

### 꼬리질문 대비

| 질문 | 한 줄 답 |
|------|----------|
| Confidence가 뭔가요? | 물체 존재 확률 × 박스가 실제와 겹치는 정도(IoU) |
| NMS는 왜 필요? | 같은 물체에 박스가 중복으로 잡혀서, 겹치는 것 중 점수 낮은 걸 제거 |
| Loss는? | 위치 오차 + Confidence 오차 + 분류 오차의 합 |
| mAP가 뭔가요? | 여러 IoU 임계값에서의 평균 정밀도, 탐지 모델 대표 성능 지표 |
| 과적합 판단은? | train loss는 내려가는데 val loss가 올라가면 과적합 |
| 2-stage와 차이? | 영역 제안 단계 없이 한 번에 → 속도↑ |

## 9. 참고: YOLO vs MediaPipe (혼동 주의)

| | YOLO | MediaPipe |
|---|---|---|
| 만든 곳 | Ultralytics 등 | Google |
| 핵심 | 객체 탐지 (무엇이 어디 있나) | 인체/손/얼굴 추적 특화 |
| 잘하는 것 | 아무 물체나 박스로 찾기 | 손 관절·얼굴 랜드마크·포즈 |
| 커스텀 학습 | ✅ 내 데이터로 학습 | △ 주로 사전학습된 기능 사용 |

- **책 탐지·번호판·제품 검수** → YOLO / **손동작·자세 교정·얼굴 필터** → MediaPipe
- 둘은 **함께 사용 가능** (예: YOLO로 사람 영역 찾고 → MediaPipe로 그 사람 포즈 분석). 영상 실시간이면 FPS·자원 부담 주의.
- MediaPipe의 "인사 동작" 인식: MediaPipe 자체를 학습시키는 게 아니라, MediaPipe가 뽑아준 **랜드마크 좌표**를 골라서 그 위에 별도 분류기를 규칙/학습으로 얹는 구조. (손 흔들기는 시간 흐름이 필요한 동작)

## 한 줄 결론

> 학습된 YOLO 모델은 결국 **파일 하나(`best.pt`)**다. task(detect/seg/pose/cls/obb)를 골라, epoch만큼 반복 학습하고, conf로 추론을 조절하며, `results.png`의 loss↓·mAP↑로 잘 됐는지 확인한다. 동작의 핵심은 **그리드로 책임을 나눠 한 번에 예측 → NMS로 중복 정리**.
