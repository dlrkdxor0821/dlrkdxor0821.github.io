# PID 제어기 정리 — 이론 + gogoping 적용 분석

> 제어 이론(개념)과 실제 프로젝트(gogoping) 코드 분석을 하나로 묶은 완전 합본.
> 흐름: **큰 그림 → 이론 → 실제 적용 → 의사결정 규칙**

---

## Part 0. 큰 그림 — PID는 "필수"가 아니라 "선택"

자율주행 / 로봇 제어에서 PID는 흔한 디폴트지만 **필수는 아니다.** 루프마다 맞는 도구가 다르다.

| 기법 | 한 줄 설명 | 강한 상황 |
|------|-----------|-----------|
| **PID** | 오차 비례·적분·미분 | 고정 목표 정밀 추종, 일관된 동역학, 모델 몰라도 OK |
| **MPC** | 미래 예측 + 제약 최적화 | 속도·가속·충돌 한계를 명시적으로 지켜야 할 때 |
| **LQR** | 상태공간 최적 제어 | 여러 입출력(MIMO)이 얽히고 모델이 있을 때 |
| **Pure Pursuit / Stanley** | 기하 기반 경로 추종 | 조향(횡방향) 경로 따라가기 |
| **P + 피드포워드** | 비례 + 외란 선보정 | 움직이는 타깃 추종 |
| **end-to-end 학습** | 센서→명령 직접 학습 | 모델링이 어려운 복잡 환경 |

> 핵심: **"PID를 쓸까?"가 아니라 "이 루프에 어떤 도구가 맞나?"** 가 올바른 질문.

## Part 1. PID 제어기란? (이론)

### 1-1. 기본 개념 — 피드백 제어

PID는 **목표값(setpoint)과 현재값의 차이(오차)를 보고, 그 오차를 줄이는 방향으로 출력을 계산**하는 피드백 제어기다.

```
오차  e(t) = (목표값) − (측정값)
제어  u(t) → 시스템(plant) → 측정 → 다시 오차 계산  (반복)
```

예: 목표 거리 0.5 m, 현재 1.2 m → 오차 +0.7 m → "전진하라" 명령.

### 1-2. 수식 — 세 항의 합

```
u(t) = Kp·e(t)  +  Ki·∫e(t)dt  +  Kd·de(t)/dt
        └─ P ─┘    └──── I ────┘   └──── D ────┘
```

이산(코드) 형태:

```python
error    = setpoint - measurement
integral = integral + error * dt
deriv    = (error - prev_error) / dt
u = Kp*error + Ki*integral + Kd*deriv
prev_error = error
```

### 1-3. 세 항의 직관 — "과거·현재·미래"

| 항 | 무엇을 보나 | 시간 | 역할 | 부작용 |
|----|-----------|------|------|--------|
| **P** | 지금 오차 | 현재 | 오차에 비례해 즉시 반응 | 정상상태 오차 잔존, 크면 진동 |
| **I** | 누적된 오차 | 과거 | 남은 정상오차를 0으로 메움 | windup, 위상지연 → 불안정화 |
| **D** | 오차의 변화율 | 미래(예측) | 빠르게 다가오면 미리 제동(오버슈트 억제) | 노이즈 증폭 |

**비유 (차를 목표 지점에 세우기):**
- **P** — "멀수록 세게 밟기"
- **I** — "언덕(중력) 때문에 자꾸 못 미치면 조금씩 더 밟기"
- **D** — "곧 도착하니 미리 브레이크"

### 1-4. 튜닝 (게인 잡기)

핵심은 **P부터 시작해 필요한 항만 추가**:

```
1) Ki=Kd=0 두고 Kp 키우기 → 반응 빠르지만 진동 직전까지
2) 정상상태 오차 남으면 → Ki 추가 (천천히)
3) 오버슈트/진동 거슬리고 센서 깨끗하면 → Kd 추가
```

(고전적으로 Ziegler–Nichols 공식도 있지만, 현장은 위처럼 손으로 잡는 경우가 많다.)

### 1-5. 흔한 변형

- **PI** — D를 빼서 노이즈에 강함. 산업 루프의 다수.
- **PD** — I를 빼서 windup 없음. 위치 제어 등.
- **anti-windup** — 막혔을 때 적분기 누적 차단 (I 쓸 때 거의 필수)
- **derivative filter** — D 앞에 저역통과 필터 (노이즈 완화)

## Part 2. gogoping 적용 분석 (실제 코드)

### 2-1. 실제로 쓴 것 = P (순수 비례 제어)

추종 제어 핵심 `reactive_control.py`:

```python
linear_x  = KP_LIN * (distance - target_distance)   # KP_LIN = 0.5
angular_z = KP_ANG * angle_error                     # KP_ANG = 0.6
```

- **I항 없음, D항 없음** → PID도 PD도 아닌 **순수 P**
- docstring부터 `"Moca-style reactive P-control"`
- ⚠️ 문서(`tech-stack.md`)엔 "PD/PID"라 적혀 있지만 **실제 코드는 P** — 문서가 느슨했던 것

### 2-2. 왜 P만? — 각 항을 *일부러* 뺀 이유 (유치원 = 안전 우선)

| 뺀 항 | 뺀 이유 | 대신 쓴 것 |
|-------|---------|-----------|
| **I (적분)** | **windup 위험.** STOP·safety_stop·lost·doorway 등 멈춤이 잦음 → 적분기에 오차 누적 → 풀리는 순간 **아이 쪽으로 돌진.** 코드도 `"멈출 땐 즉시 0, 출발은 0부터"` (상태 없는 P라 공짜) | 데드밴드 (작은 오차 무시) |
| **D (미분)** | **노이즈 증폭.** 거리/각도가 ReID+depth+LiDAR라 지터 심함 → cmd_vel 발작 | EMA 필터 + slew(가속) 제한 |

**추가 근거:**
- 목표가 **움직이는 타깃**(걷는 선생님)이라 없앨 정적오차가 없음
- **속도 명령**(Twist)이라 모터 내부 루프가 이미 닫혀 있음
- P는 **단위테스트로 정확히 검증** 가능 → 예측 가능 = 안전

### 2-3. 단 한 곳, PI를 썼어야 했던 지점 — RECOVERY body turn

yaw가 두 종류였고 답이 정반대다:

| | ① 추종 yaw | ② 탐색 body turn |
|---|---|---|
| 목표 | 사람 중앙 유지 (**움직임**) | "정확히 25° 회전" (**고정 각도**) |
| 피드백 | 비전 폐루프 ✅ | **없음 (개루프 dead reckoning)** |
| 적정 제어 | **P** (지금이 맞음) | **PI** (지금은 부족) |

`consume_body_turn`이 `remaining -= rate×dt`로 "돌았겠지" 가정 → 실제 좌우 비대칭 때문에 `CHASSIS_TURN_SCALE_RIGHT = 0.7` **매직 상수로 손튜닝**.

이게 바로 **I항이 자동으로 해줄 일**(계통오차 제거)을 수동으로 한 흔적. yaw odom/IMU로 **폐루프 PI**를 걸었으면 `0.7` 없이 정확·견고했을 것.

> `0.7` = 피드포워드(아는 외란 선보정), 빠진 것 = I항(잔여 외란 사후 보정). 정석은 **둘을 합치는 것.**

### 2-4. "그냥 PID 쓰면 좋은 거 아냐?" → No, 공짜가 아니다

각 항은 **새 고장 모드를 같이 들고 온다:**
- **I** → windup, 위상 지연 → 안정한 루프를 발산시킬 수도, 숨은 상태, anti-windup 로직 부담
- **D** → 고주파 노이즈 증폭, derivative kick

그리고 **튜닝 안 된 PID < 잘 고른 P.** 게인 3개가 간섭해서 대충 넣으면 진동/발산. 현장 실상도 대부분 **PI·P**, 풀 PID는 소수파. 성숙도는 항을 **더하는** 게 아니라 **깎아내는** 쪽 — 특히 안전 환경에선 **단순함 = 안전.**

## Part 3. 의사결정 규칙 (가져가실 한 장)

```
P     → 기본. 무조건 P부터.

I 켜라 → 없애야 할 "정상상태 오차"가 실제로 있고 + 고정 setpoint일 때만
          (온도/정속/고정각도 도달 · gogoping body turn ✅)
          (움직이는 타깃 ❌ — gogoping 추종)

D 켜라 → 오버슈트가 비싸고 + 센서가 깨끗할 때만
          (IMU/엔코더 ✅ / 비전·거친 센서 ❌ → 대신 EMA+slew)

제약(속도·충돌)을 명시적으로 지켜야 → PID 졸업, MPC 고려
```

## 한 줄 요약

> **gogoping은 PID가 아니라 P를 썼고, 안전 환경에선 그게 맞는 선택이었다.** PID는 "고급 P"가 아니라 각 항(I·D)이 자기 비용을 내고 들어오는 도구상자 — 해결할 문제(정적오차/오버슈트)가 실제로 있는 항만 켜는 게 정석. 전 시스템에서 PI가 진짜 이득이었을 곳은 개루프로 처리한 **body turn 하나**뿐이었다.
