# PBVS vs IBVS — 비주얼 서보잉

카메라로 목표를 보면서 팔을 실시간으로 정렬하는 **폐루프 제어**. "지금 상태 vs 원하는 상태"의 **오차를 어느 공간에서 재느냐** — 3D냐 이미지냐 — 로 두 방식이 갈린다.

---

## 비주얼 서보잉이란

한 번 계산하고 끝(open-loop)이 아니라, **카메라 → 오차 계산 → 팔 이동 → 다시 카메라**를 반복하는 피드백 루프다. 캘리브레이션이 조금 틀려도 매 프레임 오차를 보며 수정하므로 정밀 정렬에 강하다. 오차 `e`가 0에 수렴하면 정렬 완료.

![비주얼 서보잉 폐루프 — 카메라→특징/포즈→오차→제어기→로봇→다시 카메라](https://images.prismic.io/asd0821/k3OoPxx3FTP9831R_vs-loop.png?auto=format,compress)

---

## 핵심 차이 — 오차를 어디서 재나

### PBVS (Position-Based) — 3D 직교 공간에서

**파이프라인**
1. 이미지에서 특징/알려진 객체 검출
2. **포즈 추정(PnP)** — 목표의 3D 위치·자세 복원
3. 현재 포즈 ↔ 목표 포즈의 **3D 오차** 계산
4. 제어기가 **직교 속도** 출력 → IK로 관절 이동

**오차·제어:** `e = (t, θu)`, `v_c = −λ · e`

### IBVS (Image-Based) — 2D 이미지 공간에서

**파이프라인**
1. 이미지에서 특징점(픽셀 좌표) 추출 → s
2. 목표 특징점 s* 와 **픽셀 오차** `e = s − s*` 계산
3. **이미지 자코비안** Lₛ 로 픽셀 오차 → 카메라 속도 변환
4. 제어 속도로 로봇 이동 (3D 복원 없이)

**오차·제어:** `ṡ = Lₛ · v_c` → `v_c = −λ · Lₛ⁺ · e`

---

## 한눈에 비교

| 기준 | PBVS | IBVS |
| ---- | ---- | ---- |
| 오차 재는 공간 | 3D 직교 (포즈) | 2D 이미지 (픽셀) |
| 3D 복원 | **필요** (PnP) | 불필요 |
| 제어 변수 | 포즈 오차 | 특징점 오차 s − s* |
| 핵심 도구 | 포즈 추정 + IK | **이미지 자코비안 Lₛ** |
| 강점 | 곧은 3D 경로, 직관적 | 캘리브 오차에 강건, FOV 유지 |
| 약점 | 캘리브·FOV 이탈에 취약 | 경로 왜곡, 깊이 의존, 지역 최소 |
| 연결 | 끝단 좌표 제어 · IK | 자코비안 · 인지(특징 검출) |

---

## 깊이 짚기 — 이미지 자코비안 Lₛ

카메라 속도 `v_c` 와 이미지 위 특징점 속도 `ṡ` 를 잇는 행렬(`ṡ = Lₛ·v_c`). 앞서 본 로봇 자코비안과 **똑같은 발상**을 "관절↔손끝"이 아니라 "카메라↔픽셀"에 적용한 것. 제어법 `v_c = −λ Lₛ⁺ e` 로 오차를 지수적으로 줄인다.

점 특징(정규화 좌표 x, y · 깊이 Z)의 이미지 자코비안 — 실제로 코딩하는 행렬:

```
      ⎡ −1/Z    0     x/Z     x·y    −(1+x²)    y ⎤
Lₛ =  ⎢                                            ⎥
      ⎣   0   −1/Z    y/Z    1+y²     −x·y     −x ⎦

카메라 속도 v_c = (v_x, v_y, v_z, ω_x, ω_y, ω_z)  →  픽셀 속도 (ẋ, ẏ)
```

행렬 안에 **깊이 Z**가 직접 박혀 있다 — IBVS가 Z 추정에 의존하는 이유가 수식으로 그대로 보인다. 특징점 하나당 이 2행이 쌓여 전체 Lₛ 가 만들어진다(점 3개면 6×6).

**PBVS 오차 정의** — 현재 포즈에서 목표 포즈까지의 **병진 t** 와 **회전(축·각) θu** 를 묶어 `e = (t, θu)`, 제어법은 `v_c = −λ·e`. 오차가 3D 물리량이라 병진·회전이 깔끔히 분리(decouple)돼 **TCP가 곧은 경로**로 수렴한다.

### 주요 함정 / 개념

- **깊이(Z) 의존성 — IBVS의 아킬레스건** — Lₛ 안에 각 특징점의 깊이 Z가 들어간다. Z를 정확히 모르면 근사값을 쓰는데, 이 때문에 수렴이 흔들리거나 경로가 왜곡된다. (그래서 depth 카메라가 도움)
- **FOV 이탈 — PBVS의 약점** — PBVS는 3D 오차만 보고 움직이므로 도중에 목표가 화면 밖으로 나가 추적을 잃을 수 있다. IBVS는 픽셀을 직접 제어해 특징을 화면 안에 붙잡아 둔다.
- **하이브리드 (2.5D VS)** — 회전은 3D 포즈로, 병진은 이미지 특징으로 나눠 제어 → 곧은 경로 + FOV 유지를 동시에 노린다.

## 카메라 배치 & 실무 스택

| 배치 | 구성 | 특징 |
| ---- | ---- | ---- |
| **Eye-in-hand** | 카메라가 팔 끝단에 부착 | 손끝 시점, 정밀 정렬에 유리 · 카메라도 팔과 함께 움직임 |
| **Eye-to-hand** | 카메라가 외부에 고정 | 전체 장면 조망 · 팔이 스스로 시야를 가릴 수 있음 |

두 배치는 **핸드-아이 캘리브레이션**(카메라↔로봇 좌표 변환)이 자코비안에 어떻게 들어가는지가 달라진다. 실무에선 밑바닥부터 짜기보다 **ViSP**(Visual Servoing Platform)·**MoveIt**·**OpenCV**(특징검출·PnP)를 쓰고, 제어 게인 `λ` 튜닝, 이미지 **지연(latency)**, 특징 추적 끊김 처리가 실제 난제다.

---

> **실전 감각:** 카메라 보정·객체 모델이 정확하면 **PBVS**(곧은 경로), 그게 불안하거나 특징을 화면에 붙잡아야 하면 **IBVS**(강건). 실무에선 **하이브리드**가 많다. 어느 쪽이든 앞단엔 특징 검출·포즈 추정(인지)이, 뒷단엔 **자코비안·IK**(제어)가 붙는다. ([[순기구학 vs 역기구학 (FK vs IK)]] 참고)
