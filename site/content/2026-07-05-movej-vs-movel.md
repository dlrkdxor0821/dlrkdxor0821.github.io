# MoveJ vs MoveL — 로봇팔 모션 명령

로봇팔에게 "여기서 저기로 가"라고 시키는 두 가지 방식. 무엇을 기준으로 움직이느냐 — **관절 각도냐, 손끝 직선이냐** — 가 갈린다.

![MoveJ 곡선 경로 vs MoveL 직선 경로 — 같은 두 점 A→B 이동 비교](https://images.prismic.io/asd0821/OPx820j96RKu2-nM_movej-movel-path.png?auto=format,compress)

---

## 개념

| 명령 | 이름 | 기준 | 한 줄 |
| ---- | ---- | ---- | ----- |
| **MoveJ** | Joint Move (관절 공간) | 조인트 각도 | 각 관절 각도를 시작→목표로 보간. "팔 각도를 이만큼 돌려" |
| **MoveL** | Linear Move (직교 공간) | TCP 좌표 | TCP가 직선으로 이동. "손끝을 A에서 B까지 곧게" |

## 장단점

| | MoveJ | MoveL |
| ---- | ----- | ----- |
| **장점** | 빠르고 부드럽다 · 특이점에 안전 · 연산이 가볍다 | 경로 예측 가능(직선) · 정밀 접근·조립에 적합 |
| **단점** | TCP 경로 예측 불가(곡선) · 좁은 공간·장애물 근처에서 위험 | 느리다 · 특이점·관절 한계에 취약 · 매 스텝 IK 연산 부담 |

---

## 한눈에 비교

| 기준 | MoveJ | MoveL |
| ---- | ----- | ----- |
| 기준 공간 | 관절 각도 | **직교 좌표 (TCP)** |
| TCP 경로 | 곡선 (예측 X) | **직선** |
| 속도 | **빠름** | 느림 |
| 특이점(Singularity) | 안전 | 걸릴 수 있음 |
| 연산 | 관절 보간 | 매 순간 역기구학(IK) |
| 용도 | 자유 이동 | 정밀 접근·직선 작업 |

---

## 사용 용도

| MoveJ — 경로보다 속도 | MoveL — 경로가 곧 품질 |
| --------------------- | ---------------------- |
| 홈 포지션 복귀 · 대기 자세로 이동 | 물체로 수직 접근·후퇴 (approach / retract) |
| 픽 지점 → 플레이스 지점 사이의 큰 이동 | 삽입·조립 (peg-in-hole), 곧게 넣고 빼기 |
| 장애물 없는 공간을 빠르게 가로지를 때 | 용접·도포·디스펜싱 등 표면 직선 경로 |
| 카메라 촬영 위치 등 자세만 맞추면 되는 이동 | 팔레타이징에서 곧게 내려놓기, 컨베이어 추종 |

---

> **실전 감각:** 책 근처까지는 **MoveJ**로 빠르게 이동 → 마지막에 책으로 곧게 들어갈 땐 **MoveL**. 필요하면 원호를 그리는 `MoveC`도 있다.
