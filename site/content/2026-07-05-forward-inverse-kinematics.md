# 순기구학 vs 역기구학 (FK vs IK)

기구학은 **조인트 각도**와 **손끝(TCP) 위치** 사이를 오가는 변환이다. 어느 방향으로 푸느냐에 따라 난이도가 완전히 갈린다.

![관절 공간(θ)과 직교 공간(x,y,z) 사이 FK/IK 변환 방향](https://images.prismic.io/asd0821/DT8gvavY_ZaevVRO_fk-ik-map.png?auto=format,compress)

---

## FK / IK 개념

| | FK (순기구학) | IK (역기구학) |
| ---- | ------------- | ------------- |
| 이름 | Forward Kinematics | Inverse Kinematics |
| 입력 | 조인트 각도 (θ₁, θ₂ …) | 원하는 TCP 위치·자세 |
| 출력 | TCP 위치·자세 (x, y, z + 방향) | 조인트 각도 조합 |
| 질문 | "관절을 이만큼 돌리면 손끝이 **어디에**?" | "손끝을 **저기**에 두려면 관절을 얼마나?" |
| 해 | **유일해** (항상 1개) | **다중해** · 없을 수도 있음 |
| 난이도 | 쉬움 (곱셈으로 전개) | 어려움 (역문제) |
| 쓰이는 곳 | 현재 자세 계산·시뮬레이션 | **목표 좌표로 실제 이동** (MoveL 등) |

---

## 왜 IK가 어려운가

1. **해가 여러 개다 (다중해)** — 같은 손끝 위치를 팔꿈치 위(elbow-up)/아래(elbow-down) 등 여러 자세로 도달 가능. 로봇은 그중 하나를 골라야 한다.
2. **해가 없을 수도 있다** — 목표가 팔이 닿는 범위(**워크스페이스**) 밖이면 어떤 각도 조합으로도 도달 불가.
3. **특이점에서 무너진다** — 특이점(Singularity)에서 자유도가 줄어 해가 무한하거나 각속도가 발산 → 계산·제어 불안정.

---

## IK 푸는 방법

| 방식 | 해석적 (Closed-form) | 수치적 (Numerical) |
| ---- | -------------------- | ------------------ |
| 원리 | 기하·삼각함수로 수식 직접 유도 | 자코비안으로 반복 근사 (Newton 계열) |
| 장점 | 빠르고 정확, 실시간 유리 | 구조 제약 없이 범용 적용 |
| 단점 | 특정 구조(구형 손목 등)만 가능 | 느리고 초기값·특이점에 민감 |

---

## 자코비안 (Jacobian)

수치적 IK의 엔진. **관절이 움직이는 속도**와 **손끝이 움직이는 속도**를 잇는 행렬이다.

```
ẋ = J · θ̇        (관절 속도 → TCP 속도, 순방향·쉬움)
θ̇ = J⁻¹ · ẋ      (원하는 손끝 속도 → 관절 속도, 역방향)
```

θ̇ = J⁻¹·ẋ 를 조금씩 반복하는 게 수치적 IK다.

**구조** — 행 = TCP 자유도(보통 6 = 위치 3 + 자세 3), 열 = 관절 수. 6축 로봇이면 6×6 정방행렬이라 역행렬을 바로 구할 수 있다.

**용도**

| 용도 | 식 | 설명 |
| ---- | -- | ---- |
| 속도 IK | `θ̇ = J⁻¹·ẋ` | 목표 방향으로 조금씩 이동시키며 반복 → 수치적 IK의 핵심 |
| 힘·토크 매핑 | `τ = Jᵀ·F` | 손끝에 주려는 힘 F ↔ 필요한 관절 토크 τ (정역학) |
| 특이점 판별 | `det J = 0` | 행렬식이 0이면 특이점 — J가 역행렬을 잃는 지점 |

**특이점에서 발산** — 특이점에서 J는 랭크를 잃어 J⁻¹가 폭발 → 관절 속도가 무한대로 튄다. 물리적으로 한 방향의 자유도를 잃은 상태.

**여유 자유도 (7축 이상)** — 관절이 6보다 많으면 J가 정방행렬이 아니라 역행렬이 없다. 이땐 **유사역행렬(pseudo-inverse) J⁺**로 "가장 그럴듯한" 해를 구한다.

---

## 제어 방식과의 연결

FK/IK는 *변환*이고, 관절/끝단 제어는 *제어 방식*이다. **어느 공간에서 명령하느냐**가 IK를 쓰는지를 결정한다.

| | 관절값으로 제어 | 끝단 좌표로 제어 |
| ---- | --------------- | ---------------- |
| 공간 | 관절 공간 (Joint space) | 직교 공간 (Cartesian) |
| 명령 | 조인트 각도 직접 ("1번 30°, 2번 45°") | TCP 좌표 ("손끝을 x, y, z로") |
| 변환 | **불필요** (FK는 결과 위치 참고용) | **IK 필수** — 내부에서 각도로 변환 |
| 대응 명령 | MoveJ | MoveL |

> 핵심: **끝단 좌표 제어일 때만 IK가 돌아간다.** 관절 제어는 변환 없이 각도를 바로 명령한다. (FK는 제어에 꼭 필요한 게 아니라 "지금 각도면 손끝이 어디?"를 알려주는 참고 도구.)

---

> **실전 감각:** "이 좌표로 가"(`MoveL`)를 로봇이 실행하려면 결국 **IK를 풀어** 각 관절 각도를 구해야 한다. **MoveL이 느린 이유**가 바로 이것 — 경로 위 매 순간 IK를 반복해서 풀기 때문. ([[MoveJ vs MoveL]] 참고)
