# DDS 완전 정리 (ROS2)

## 1. DDS란? — 개념

ROS2는 여러 프로그램(노드)이 데이터를 주고받는다. 이 **"어떻게 주고받느냐"** 를 담당하는 통신 토대가 **DDS(Data Distribution Service)** 다.

**비유 — 단체 카톡방 📱**

- 노드 = 사람들 / 토픽 = 채팅방 / DDS = 채팅 시스템 자체

**ROS1 vs ROS2**

- ROS1: 중간에 **총무(master)** 가 연결 → 총무 죽으면 다 끊김 ❌
- ROS2(DDS): 총무 없이 **노드끼리 알아서 찾아 직접 대화** ✅

> **DDS = ROS2 노드들이 총무 없이 서로 찾아 직접 통신하게 해주는 표준 통신 방식.**

## 2. DDS의 3대 동작 원리

| 원리 | 뜻 |
|------|-----|
| **Pub/Sub (발행/구독)** | 보내는 쪽(publisher)이 토픽에 던지면, 듣는 쪽(subscriber)이 받음 |
| **Discovery (자동 탐색)** | 총무 없이 노드끼리 "나 여기 있어!" 외쳐서 서로 발견 |
| **QoS (품질 정책)** | 토픽마다 "확실히 전달 vs 빠르게 대충" 등 전달 품질 설정 |

---

## 3. ROS2 통신 3종류 — 전부 DDS Pub/Sub 위에서 돈다

DDS가 원래 제공하는 건 **오직 Pub/Sub(토픽)**. Service·Action은 그걸 **조합해 만든 상위 패턴**이다.

| 방식 | 모양 | 비유 | 리비 예시 |
|------|------|------|-----------|
| **Topic** | 계속 흘려보냄, 다대다 | 라디오 📻 | 카메라·라이다, `/cmd_vel` |
| **Service** | 요청→응답, 1:1 | 전화 ☎️ | "배터리 몇%?" 묻고 답 |
| **Action** | 긴 작업+진행+취소 | 택배 📦 | "서가 가서 책 집기" |

```
Topic   = DDS Pub/Sub 그 자체
Service = 요청용 토픽 + 응답용 토픽
Action  = Service(목표·취소·결과) + Topic(피드백·상태)
```

> 즉 **모든 ROS2 통신은 밑바닥에서 DDS Pub/Sub로 굴러간다.**

### 언제 뭘 쓰나

| 상황 | 선택 |
|------|------|
| 데이터를 계속 흘려보냄 (센서, 속도) | **Topic** |
| 한 번 묻고 바로 답 받기 (설정 변경, 상태 조회) | **Service** |
| 오래 걸리는 작업 + 진행상황 보고 + 중간 취소 | **Action** |

## 4. DDS 종류 — Fast DDS vs Cyclone DDS

DDS는 **규격(약속)** 이라 만든 회사가 여러 곳이다. 하는 일은 같고 **만든 방식만 다르다** (카톡 vs 라인).

### 1️⃣ Fast DDS (eProsima) — 기본값

- **원리**: 표준 DDS를 풀스펙 구현 + 추가 기능. 같은 PC 노드끼린 **공유메모리(Shared Memory)** 로 직행 (네트워크 안 거침 → 빠름)
- **기능**: 공유메모리(대용량 초고속), Discovery Server(대규모 탐색 최적화), XML 세밀 설정
- **언제**: 그냥 시작 / 한 PC에서 영상·라이다 대용량 / 노드 수백 개 대규모

### 2️⃣ Cyclone DDS (Eclipse) — 가볍고 튼튼

- **원리**: 꼭 필요한 것만 단순 구현 → 가볍고 동작이 **예측 가능, 잘 안 끊김**
- **기능**: 경량(저메모리), 안정적·결정적, 설정 간단(`CYCLONEDDS_URI`)
- **언제**: **멀티로봇** / 통신 꼬일 때 1순위 전환 / 라즈베리파이 등 저사양 / Nav2 불안정

### 선택 가이드

| 상황 | 선택 |
|------|------|
| 그냥 시작 / 단일 로봇 | **Fast DDS** |
| 한 PC 대용량(영상·PCL) | **Fast DDS** (공유메모리) |
| 노드 수백 개 | **Fast DDS** (Discovery Server) |
| **멀티로봇 / RMF** | **Cyclone DDS** |
| 통신 꼬임·끊김 디버깅 | **Cyclone DDS** |
| 저사양 보드(파이) | **Cyclone DDS** |

### 바꾸는 법 + 주의

```bash
sudo apt install ros-jazzy-rmw-cyclonedds-cpp
export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp   # Cyclone
export RMW_IMPLEMENTATION=rmw_fastrtps_cpp     # Fast(기본)
```

> ⚠️ **모든 노드가 같은 종류여야 통신됨** (Fast ↔ Cyclone 섞으면 대화 안 됨)

## 한 줄 요약

> **DDS** = ROS2가 총무 없이 직접 통신하는 토대(Pub/Sub·Discovery·QoS).
> **통신 3종**(Topic·Service·Action)도 전부 DDS Pub/Sub 위에서 동작.
> **종류**는 Fast DDS(기본·기능많음)와 Cyclone DDS(가볍고 튼튼) — 멀티로봇/꼬일 때 Cyclone, 섞으면 안 됨.
