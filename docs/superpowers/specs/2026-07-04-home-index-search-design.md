# 홈 화면 카테고리 정렬 + 사이드바 상시 검색

## 배경

홈(`/`)은 글 총 개수만 보여주고 목록이 없어 "뭐가 있는지 찾기 어렵다"는 문제가 있었다.
1차로 홈을 검색창+필터+전체 리스트로 바꿔봤는데, 브라우저 목업으로 다시 비교한 뒤 다음으로 확정했다:

- **검색은 사이드바 맨 위에 항상 떠있는 입력창** (모든 페이지에서 상시 노출, 클릭해서 여는 오버레이 아님)
- **홈은 검색/필터 UI 없이 Project/Study 카테고리별로 글 목록만 깔끔하게 정렬**

## 목표

1. 사이드바 상단(브랜드 아래)에 검색 입력창을 상시 노출한다. 입력하면 사이드바 트리 자체가
   제목/태그/카테고리명 기준으로 필터링되어, 매칭되는 글만 남고 매칭 없는 카테고리는 숨겨진다.
2. 홈페이지는 Project/Study 대분류 아래 각 카테고리와 그 카테고리의 글 목록을 바로 보여준다
   (검색창·필터 칩 없음 — 검색은 사이드바가 전담).
3. 글이 없는 카테고리는 홈 목록에서 생략한다 (사이드바는 기존처럼 0개 카테고리도 표시 유지).

## 데이터 흐름 / 컴포넌트 경계

- `app/components/Sidebar.tsx`: 기존 Project/Study 트리에 검색 상태(`useState`)를 추가.
  쿼리가 있으면 각 카테고리의 `logs`를 제목/슬러그 기준으로 필터링하고, 결과가 0개인 카테고리는
  렌더링하지 않는다. 검색 로직은 사이드바 내부에 자체 포함 — 외부에서 별도 상태를 주입하지 않는다.
- `app/page.tsx`: 기존처럼 `getProjects()` + `getPostsByProject()`로 카테고리별 글 목록을 만들어
  Project/Study로 그룹핑해 렌더링한다. 서버 컴포넌트로 충분 — 클라이언트 상태 불필요하므로
  1차에서 만든 `HomeIndex.tsx`(검색+필터 클라이언트 컴포넌트)는 걷어내고 단순 목록 렌더링으로 교체한다.

## 비목표

- 홈페이지에 검색/필터 UI 없음 (사이드바로 대체)
- 검색은 여전히 제목/태그/카테고리명까지만 (본문 전체 검색 없음)
- 검색 결과 오버레이/모달 없음 — 사이드바 트리가 곧 검색 결과

## 후속 수정

- `site/scripts/publish.sh`의 `rsync --delete`가 `docs/`, `.superpowers/`를 제외하지 않아
  빌드 결과물 배포 때마다 스펙 문서가 삭제되는 버그를 발견 — exclude 목록에 추가해 수정함.
