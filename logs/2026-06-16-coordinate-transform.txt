2:T252e,<h1>좌표계 변환 정리 — 회전·이동·동차변환과 쿼터니언</h1>
<blockquote>
<p><strong>좌표계 변환은 &quot;한 좌표계에서 표현된 점/방향을 다른 좌표계 기준으로 다시 표현&quot;하는 수학이다.</strong>
흐름: <strong>왜 → 이동·회전 → 동차변환 행렬 → 합성(체이닝) → 오일러 vs 쿼터니언 → ROS 코드 → 규약</strong>
(ROS2의 <em>구현</em>은 [[2026-06-16-tf]], 이 노트는 그 <em>수학적 기반</em>)</p>
</blockquote>
<hr>
<h2>Part 0. 왜 좌표계 변환을 하는가</h2>
<p><strong>모든 센서와 부품이 &quot;각자 자기 기준(좌표계)&quot;으로 정보를 말하기 때문이다.</strong> 그대로는 서로 합치거나 비교하거나 행동에 쓸 수 없어서, <strong>같은 기준으로 번역</strong>해주는 게 좌표계 변환이다.</p>
<h3>① 센서마다 측정 기준이 다르다</h3>
<p>LiDAR는 &quot;LiDAR 원점&quot; 기준, 카메라는 &quot;카메라&quot; 기준으로 좌표를 말한다.</p>
<pre><code>LiDAR:  &quot;내 앞 2m에 벽&quot;           ← LiDAR 좌표
필요:   &quot;지도상 (5.3, 1.2)의 벽&quot;   ← map 좌표
</code></pre>
<p>로봇이 그 정보로 <strong>판단·행동하려면</strong> 공통 기준(보통 <code>map</code> 또는 <code>base_link</code>)으로 바꿔야 한다.</p>
<h3>② 센서가 로봇 중심에서 떨어져 달려 있다</h3>
<p>LiDAR가 로봇 중심보다 30cm 앞·20cm 위에 붙어 있으면, &quot;LiDAR 앞 2m&quot;는 실제론 &quot;로봇 중심 앞 2.3m&quot;다. 이 <strong>물리적 장착 오프셋</strong>을 보정해야 충돌 계산이 맞는다. (= <code>base_link→laser</code> 변환)</p>
<h3>③ 여러 센서를 융합하려면 같은 좌표여야 한다</h3>
<p>LiDAR + 카메라 + IMU를 합칠 때 모두 공통 프레임으로 모아야 같은 물체가 같은 위치로 보인다. 기준이 다르면 같은 벽이 두 군데로 보여 융합 불가.</p>
<h3>④ 정보가 사는 좌표계 ≠ 행동이 일어나는 좌표계</h3>
<ul>
<li><strong>경로 계획</strong>은 <code>map</code>(고정 지도) 기준</li>
<li><strong>모터 명령</strong>은 <code>base_link</code>(로봇 몸체) 기준 — &quot;앞으로&quot;, &quot;왼쪽으로 회전&quot;</li>
<li>목표는 지도 좌표인데 바퀴는 로봇 기준으로 도니까 → <strong>변환이 둘을 잇는다</strong></li>
</ul>
<h3>한 미션으로 보는 흐름</h3>
<p>&quot;지도상 (5, 1)로 가라&quot; 한 번에도 좌표계가 계속 바뀐다:</p>
<pre><code>목표 (map 좌표)
   │  ← 변환
LiDAR가 본 장애물 (laser 좌표) → 충돌 판단 (map/odom 좌표)
   │  ← 변환
&quot;왼쪽으로 0.3rad 돌며 0.2m/s&quot; (base_link 좌표) → 모터
</code></pre>
<blockquote>
<p>즉 <strong>&quot;서로 다른 언어로 말하는 부품들을 하나의 공용어로 통역&quot;</strong> 하는 작업. 이 모든 &quot;← 변환&quot;을 자동 처리하는 게 [[2026-06-16-tf]]의 TF 트리이고, 그 변환을 곱셈으로 잇는 수학이 아래의 동차변환 행렬이다.</p>
</blockquote>
<hr>
<h2>Part 0-1. 변환의 두 구성 요소</h2>
<p>변환은 두 가지의 조합이다:</p>
<pre><code>변환(Transform) = 이동(Translation) + 회전(Rotation)
</code></pre>
<hr>
<h2>Part 0-2. 좌표계 변환하는 법 — 플로우</h2>
<p>점/방향 하나를 다른 좌표계로 바꾸는 <strong>실제 과정</strong>:</p>
<pre><code>1. 출발 좌표계 확인   어느 프레임의 점인가?        예: laser 기준 (2, 0)
        ▼
2. 도착 좌표계 결정   어디 기준으로 바꿀까?         예: map
        ▼
3. 변환 T 확보       T_출발→도착 = 이동 + 회전      (Part 1~2)
                    여러 프레임 거치면 경로상 변환을 곱  (Part 3)
                    T_laser→map = T_laser→base · T_base→odom · T_odom→map
        ▼
4. 동차좌표화        점 끝에 1 붙이기             p = [x, y, z, 1]ᵀ   (Part 2)
        ▼
5. 행렬 곱           p_도착 = T · p_출발
        ▼
6. 결과 해석         도착 좌표계 기준 좌표 획득      예: map 기준 (5.3, 1.2)
</code></pre>
<blockquote>
<p><strong>한 줄 방법</strong>: <em>&quot;출발↔도착 두 프레임의 변환행렬 T(이동+회전)를 구해 점에 곱하면 끝 — 여러 프레임을 거치면 경로상의 T들을 차례로 곱해서 잇는다.&quot;</em></p>
</blockquote>
<p>ROS 실무에서는 이 1~6단계를 직접 안 하고 TF가 자동 처리한다 ([[2026-06-16-tf]]):</p>
<pre><code class="language-python"># tf2가 트리에서 T_laser→map을 찾아 곱셈까지 대신 해줌
p_map = tf_buffer.transform(p_laser, &quot;map&quot;)
</code></pre>
<hr>
<h2>Part 1. 이동과 회전</h2>
<h3>이동 (Translation)</h3>
<p>원점을 옮기는 것. 단순 덧셈: <code>p&#39; = p + t</code> (t = [tx, ty, tz]).</p>
<h3>회전 (Rotation)</h3>
<p>축을 돌리는 것. 2D에서 θ만큼 회전:</p>
<pre><code>R(θ) = | cosθ  −sinθ |
       | sinθ   cosθ |
p&#39; = R(θ) · p
</code></pre>
<ul>
<li><strong>회전행렬 R</strong>: 직교행렬, <code>Rᵀ = R⁻¹</code>, det(R)=1. 곱하는 순서가 중요(회전은 교환법칙 ✗).</li>
<li>3D는 x/y/z축 회전을 합성 (roll·pitch·yaw).</li>
</ul>
<hr>
<h2>Part 2. 동차 변환 행렬 (Homogeneous Transform)</h2>
<p>이동과 회전을 <strong>하나의 행렬</strong>로 합쳐 한 번에 곱하려는 트릭. 좌표 뒤에 1을 붙여 차원을 하나 늘린다 (동차좌표).</p>
<p>3D 변환 (4×4):</p>
<pre><code>T = | R   t |        p&#39; = T · p
    | 0   1 |
       (R: 3×3 회전, t: 3×1 이동)
</code></pre>
<p>장점: <strong>변환의 연쇄를 행렬 곱으로</strong> 표현 → 합성이 깔끔.</p>
<hr>
<h2>Part 3. 합성 (Chaining) — TF 트리의 수학</h2>
<p><code>A→C</code> 변환을 모를 때, <code>A→B</code>와 <code>B→C</code>를 알면:</p>
<pre><code>T_A→C = T_A→B · T_B→C
역변환:  T_B→A = (T_A→B)⁻¹
</code></pre>
<blockquote>
<p>바로 이게 [[2026-06-16-tf]] 트리가 <em>&quot;laser→map 줘&quot;</em> 에 답하는 방법 — 경로상의 변환들을 <strong>곱해서</strong> 만든다. (<code>map→odom→base_link→laser</code> 곱)</p>
</blockquote>
<hr>
<h2>Part 4. 방향 표현 — 오일러 vs 쿼터니언</h2>
<p>3D 회전(자세)을 표현하는 두 방식:</p>
<table>
<thead>
<tr>
<th></th>
<th>오일러 각 (Euler)</th>
<th>쿼터니언 (Quaternion)</th>
</tr>
</thead>
<tbody><tr>
<td>형태</td>
<td>roll, pitch, yaw (3개)</td>
<td>x, y, z, w (4개)</td>
</tr>
<tr>
<td>장점</td>
<td>사람이 직관적</td>
<td><strong>짐벌락 없음</strong>, 보간 안정, 수치 안정</td>
</tr>
<tr>
<td>단점</td>
<td><strong>짐벌락(Gimbal Lock)</strong>, 순서 의존</td>
<td>직관 어려움</td>
</tr>
<tr>
<td>ROS</td>
<td>입력 편의용</td>
<td><strong>내부 표준</strong> (<code>geometry_msgs/Quaternion</code>)</td>
</tr>
</tbody></table>
<blockquote>
<p><strong>짐벌락</strong>: 두 회전축이 겹쳐 자유도 하나를 잃는 현상. ROS가 자세를 쿼터니언으로 저장하는 이유. 2D 주행 로봇은 <strong>Z축 회전(yaw)만</strong> 중요하므로 쿼터니언이 단순해진다.</p>
</blockquote>
<hr>
<h2>Part 5. ROS 코드 — yaw → 쿼터니언</h2>
<p>2D 로봇은 yaw 하나만 다루므로 변환이 간단하다 (Nav2 Python 실습 코드):</p>
<pre><code class="language-python">import math

def yaw_to_quat(yaw_rad: float):
    # 2D 평면 주행: Z축 회전만 → x=y=0
    z = math.sin(yaw_rad / 2.0)
    w = math.cos(yaw_rad / 2.0)
    return z, w   # (qx=0, qy=0, qz=z, qw=w)

# 라이브러리 사용 (권장)
from tf_transformations import quaternion_from_euler
qx, qy, qz, qw = quaternion_from_euler(0.0, 0.0, math.radians(deg))
</code></pre>
<p>적용 예 (goal_pose 자세 설정):</p>
<pre><code class="language-python">goal.pose.orientation.z = q[2]
goal.pose.orientation.w = q[3]
</code></pre>
<blockquote>
<p>각도는 항상 <strong>라디안</strong>으로 다루고, 입력만 도(degree)로 받아 변환. ([[2026-06-16-nav2-pkg]]의 goToPose에서 사용)</p>
</blockquote>
<hr>
<h2>Part 6. ROS 규약 (REP-103) — 꼭 지킬 것</h2>
<ul>
<li><strong>단위</strong>: 길이 m, 각도 rad, 시간 s.</li>
<li><strong>축 방향</strong>: x=앞(forward), y=왼쪽(left), z=위(up) — <strong>오른손 좌표계</strong>.</li>
<li><strong>회전 부호</strong>: 오른손 법칙 (반시계 +).</li>
<li><strong>지리 좌표</strong>: ENU (East-North-Up).</li>
</ul>
<blockquote>
<p>이 규약을 어기면 로봇이 거꾸로 돌거나 좌우가 뒤집힌다. 프레임을 새로 정의할 때 반드시 따를 것.</p>
</blockquote>
<hr>
<h2>한 줄 요약</h2>
<blockquote>
<p><strong>좌표계 변환 = 이동 + 회전이며, 동차변환 행렬(4×4)로 묶으면 변환의 연쇄가 행렬 곱이 된다 — 이것이 TF 트리가 두 프레임 사이 변환을 계산하는 원리다.</strong> 3D 자세는 짐벌락 없는 <strong>쿼터니언</strong>으로 표현하고(ROS 내부 표준), 2D 주행은 yaw 하나만 쿼터니언으로 바꾸면 된다. 단위·축은 REP-103(m·rad, x앞/y좌/z위, 오른손) 규약을 지킨다.</p>
</blockquote>
