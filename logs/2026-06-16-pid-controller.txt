<h2>Part 1. PID 제어기란? (이론)</h2>
<h3>1-1. 기본 개념 — 피드백 제어</h3>
<p>PID는 <strong>목표값(setpoint)과 현재값의 차이(오차)를 보고, 그 오차를 줄이는 방향으로 출력을 계산</strong>하는 피드백 제어기다.</p>
<pre><code>오차  e(t) = (목표값) − (측정값)
제어  u(t) → 시스템(plant) → 측정 → 다시 오차 계산  (반복)
</code></pre>
<p>예: 목표 거리 0.5 m, 현재 1.2 m → 오차 +0.7 m → &quot;전진하라&quot; 명령.</p>
<h3>1-2. 수식 — 세 항의 합</h3>
<pre><code>u(t) = Kp·e(t)  +  Ki·∫e(t)dt  +  Kd·de(t)/dt
        └─ P ─┘    └──── I ────┘   └──── D ────┘
</code></pre>
<p>이산(코드) 형태:</p>
<pre><code class="language-python">error    = setpoint - measurement
integral = integral + error * dt
deriv    = (error - prev_error) / dt
u = Kp*error + Ki*integral + Kd*deriv
prev_error = error
</code></pre>
<h3>1-3. 세 항의 직관 — &quot;과거·현재·미래&quot;</h3>
<table>
<thead>
<tr>
<th>항</th>
<th>무엇을 보나</th>
<th>시간</th>
<th>역할</th>
<th>부작용</th>
</tr>
</thead>
<tbody><tr>
<td><strong>P</strong></td>
<td>지금 오차</td>
<td>현재</td>
<td>오차에 비례해 즉시 반응</td>
<td>정상상태 오차 잔존, 크면 진동</td>
</tr>
<tr>
<td><strong>I</strong></td>
<td>누적된 오차</td>
<td>과거</td>
<td>남은 정상오차를 0으로 메움</td>
<td>windup, 위상지연 → 불안정화</td>
</tr>
<tr>
<td><strong>D</strong></td>
<td>오차의 변화율</td>
<td>미래(예측)</td>
<td>빠르게 다가오면 미리 제동(오버슈트 억제)</td>
<td>노이즈 증폭</td>
</tr>
</tbody></table>
<p><strong>비유 (차를 목표 지점에 세우기):</strong></p>
<ul>
<li><strong>P</strong> — &quot;멀수록 세게 밟기&quot;</li>
<li><strong>I</strong> — &quot;언덕(중력) 때문에 자꾸 못 미치면 조금씩 더 밟기&quot;</li>
<li><strong>D</strong> — &quot;곧 도착하니 미리 브레이크&quot;</li>
</ul>
<h3>1-4. 튜닝 (게인 잡기)</h3>
<p>핵심은 <strong>P부터 시작해 필요한 항만 추가</strong>:</p>
<pre><code>1) Ki=Kd=0 두고 Kp 키우기 → 반응 빠르지만 진동 직전까지
2) 정상상태 오차 남으면 → Ki 추가 (천천히)
3) 오버슈트/진동 거슬리고 센서 깨끗하면 → Kd 추가
</code></pre>
<p>(고전적으로 Ziegler–Nichols 공식도 있지만, 현장은 위처럼 손으로 잡는 경우가 많다.)</p>
<h3>1-5. 흔한 변형</h3>
<ul>
<li><strong>PI</strong> — D를 빼서 노이즈에 강함. 산업 루프의 다수.</li>
<li><strong>PD</strong> — I를 빼서 windup 없음. 위치 제어 등.</li>
<li><strong>anti-windup</strong> — 막혔을 때 적분기 누적 차단 (I 쓸 때 거의 필수)</li>
<li><strong>derivative filter</strong> — D 앞에 저역통과 필터 (노이즈 완화)</li>
</ul>
<hr>
<h2>Part 2. gogoping 적용 분석 (실제 코드)</h2>
<h3>2-1. 실제로 쓴 것 = P (순수 비례 제어)</h3>
<p>추종 제어 핵심 <code>reactive_control.py</code>:</p>
<pre><code class="language-python">linear_x  = KP_LIN * (distance - target_distance)   # KP_LIN = 0.5
angular_z = KP_ANG * angle_error                     # KP_ANG = 0.6
</code></pre>
<ul>
<li><strong>I항 없음, D항 없음</strong> → PID도 PD도 아닌 <strong>순수 P</strong></li>
<li>docstring부터 <code>&quot;Moca-style reactive P-control&quot;</code></li>
<li>⚠️ 문서(<code>tech-stack.md</code>)엔 &quot;PD/PID&quot;라 적혀 있지만 <strong>실제 코드는 P</strong> — 문서가 느슨했던 것</li>
</ul>
<h3>2-2. 왜 P만? — 각 항을 <em>일부러</em> 뺀 이유 (유치원 = 안전 우선)</h3>
<table>
<thead>
<tr>
<th>뺀 항</th>
<th>뺀 이유</th>
<th>대신 쓴 것</th>
</tr>
</thead>
<tbody><tr>
<td><strong>I (적분)</strong></td>
<td><strong>windup 위험.</strong> STOP·safety_stop·lost·doorway 등 멈춤이 잦음 → 적분기에 오차 누적 → 풀리는 순간 <strong>아이 쪽으로 돌진.</strong> 코드도 <code>&quot;멈출 땐 즉시 0, 출발은 0부터&quot;</code> (상태 없는 P라 공짜)</td>
<td>데드밴드 (작은 오차 무시)</td>
</tr>
<tr>
<td><strong>D (미분)</strong></td>
<td><strong>노이즈 증폭.</strong> 거리/각도가 ReID+depth+LiDAR라 지터 심함 → cmd_vel 발작</td>
<td>EMA 필터 + slew(가속) 제한</td>
</tr>
</tbody></table>
<p><strong>추가 근거:</strong></p>
<ul>
<li>목표가 <strong>움직이는 타깃</strong>(걷는 선생님)이라 없앨 정적오차가 없음</li>
<li><strong>속도 명령</strong>(Twist)이라 모터 내부 루프가 이미 닫혀 있음</li>
<li>P는 <strong>단위테스트로 정확히 검증</strong> 가능 → 예측 가능 = 안전</li>
</ul>
<h3>2-3. 단 한 곳, PI를 썼어야 했던 지점 — RECOVERY body turn</h3>
<p>yaw가 두 종류였고 답이 정반대다:</p>
<table>
<thead>
<tr>
<th></th>
<th>① 추종 yaw</th>
<th>② 탐색 body turn</th>
</tr>
</thead>
<tbody><tr>
<td>목표</td>
<td>사람 중앙 유지 (<strong>움직임</strong>)</td>
<td>&quot;정확히 25° 회전&quot; (<strong>고정 각도</strong>)</td>
</tr>
<tr>
<td>피드백</td>
<td>비전 폐루프 ✅</td>
<td><strong>없음 (개루프 dead reckoning)</strong></td>
</tr>
<tr>
<td>적정 제어</td>
<td><strong>P</strong> (지금이 맞음)</td>
<td><strong>PI</strong> (지금은 부족)</td>
</tr>
</tbody></table>
<p><code>consume_body_turn</code>이 <code>remaining -= rate×dt</code>로 &quot;돌았겠지&quot; 가정 → 실제 좌우 비대칭 때문에 <code>CHASSIS_TURN_SCALE_RIGHT = 0.7</code> <strong>매직 상수로 손튜닝</strong>.</p>
<p>이게 바로 <strong>I항이 자동으로 해줄 일</strong>(계통오차 제거)을 수동으로 한 흔적. yaw odom/IMU로 <strong>폐루프 PI</strong>를 걸었으면 <code>0.7</code> 없이 정확·견고했을 것.</p>
<blockquote>
<p><code>0.7</code> = 피드포워드(아는 외란 선보정), 빠진 것 = I항(잔여 외란 사후 보정). 정석은 <strong>둘을 합치는 것.</strong></p>
</blockquote>
<h3>2-4. &quot;그냥 PID 쓰면 좋은 거 아냐?&quot; → No, 공짜가 아니다</h3>
<p>각 항은 <strong>새 고장 모드를 같이 들고 온다:</strong></p>
<ul>
<li><strong>I</strong> → windup, 위상 지연 → 안정한 루프를 발산시킬 수도, 숨은 상태, anti-windup 로직 부담</li>
<li><strong>D</strong> → 고주파 노이즈 증폭, derivative kick</li>
</ul>
<p>그리고 <strong>튜닝 안 된 PID &lt; 잘 고른 P.</strong> 게인 3개가 간섭해서 대충 넣으면 진동/발산. 현장 실상도 대부분 <strong>PI·P</strong>, 풀 PID는 소수파. 성숙도는 항을 <strong>더하는</strong> 게 아니라 <strong>깎아내는</strong> 쪽 — 특히 안전 환경에선 <strong>단순함 = 안전.</strong></p>
<hr>
<h2>Part 3. 의사결정 규칙 (가져가실 한 장)</h2>
<pre><code>P     → 기본. 무조건 P부터.

I 켜라 → 없애야 할 &quot;정상상태 오차&quot;가 실제로 있고 + 고정 setpoint일 때만
          (온도/정속/고정각도 도달 · gogoping body turn ✅)
          (움직이는 타깃 ❌ — gogoping 추종)

D 켜라 → 오버슈트가 비싸고 + 센서가 깨끗할 때만
          (IMU/엔코더 ✅ / 비전·거친 센서 ❌ → 대신 EMA+slew)

제약(속도·충돌)을 명시적으로 지켜야 → PID 졸업, MPC 고려
</code></pre>
<hr>
<h2>한 줄 요약</h2>
<blockquote>
<p><strong>gogoping은 PID가 아니라 P를 썼고, 안전 환경에선 그게 맞는 선택이었다.</strong> PID는 &quot;고급 P&quot;가 아니라 각 항(I·D)이 자기 비용을 내고 들어오는 도구상자 — 해결할 문제(정적오차/오버슈트)가 실제로 있는 항만 켜는 게 정석. 전 시스템에서 PI가 진짜 이득이었을 곳은 개루프로 처리한 <strong>body turn 하나</strong>뿐이었다.</p>
</blockquote>
