<h3>디스패치 (SR-DSP — 현재 요구사항 문서에서 빠진 부분 ⚠️)</h3>
<ul>
<li>배정 기준 = <strong>실제 경로 ETA</strong> (직선거리 ❌, Nav2로 계산 ✅)</li>
<li>다 바쁘면 <strong>큐</strong>, 없으면 <strong>&quot;지금 불가&quot;</strong></li>
<li>RMF(경매/입찰 방식)는 표준이지만 설정이 무거움 → <strong>3대 규모면 직접 cost 함수 디스패처</strong> 추천</li>
</ul>
<pre><code class="language-python">def assign_robot(task):
    candidates = [r for r in robots if r.state == &#39;IDLE&#39;]
    if not candidates:
        task_queue.add(task)            # 다 바쁘면 큐
        return
    best = min(candidates, key=lambda r: cost(r, task))   # 비용 최소
    best.assign(task)

def cost(robot, task):
    eta = path_eta(robot.pose, task.location)   # 실제 경로 예상시간
    eta += battery_penalty(robot)
    eta -= priority_bonus(task)
    return eta
</code></pre>
<h3>⚠️ 웹↔ROS2 다리 필요</h3>
<ul>
<li>PyQt → rclpy 직접 연결 ✅</li>
<li>웹(사용자/로봇) → ROS2 DDS에 직접 못 붙음 → <strong>rosbridge</strong> 또는 <strong>FastAPI 게이트웨이</strong> 경유</li>
</ul>

<h2>2. 인식·파지 (모방학습 없이 기구학으로)</h2>

</code></pre>
<h3>깊이 얻는 법 (뎁스캠 없이도 가능)</h3>
<table>
<thead>
<tr>
<th>방법</th>
<th>적합</th>
</tr>
</thead>
<tbody><tr>
<td><strong>평면 가정 + 호모그래피</strong></td>
<td>테이블 위 픽앤플레이스 (최고)</td>
</tr>
<tr>
<td><strong>물체 실제크기 역산</strong></td>
<td>OBB 픽셀크기로 <code>Z = fx·W/w</code> 계산</td>
</tr>
<tr>
<td><strong>Eye-in-hand 모션시차</strong></td>
<td>카메라가 팔에 부착 (로봇이 베이스라인 정확히 앎, 가변 가능)</td>
</tr>
<tr>
<td>AprilTag / 스테레오</td>
<td>마커 가능 / 카메라 2대</td>
</tr>
</tbody></table>
<p><strong>Eye-in-hand 모션시차 핵심</strong>: 팔을 순수 평행이동(movel) → <code>Z = fx·B/d</code>. 깊이 오차는 <code>Z²/(fx·B)</code> 에 비례 → <strong>먼 물체일수록 많이 움직여라(B↑)</strong>. 대응점은 <code>cv2.triangulatePoints</code>, hand-eye 캘리브 필수.</p>
<h3>로봇 모션 명령</h3>
<ul>
<li><strong>movej</strong> = 관절 보간 (빠름, TCP 경로 곡선) → 자유공간 이동</li>
<li><strong>movel</strong> = TCP 직선 보간 (정밀) → 접근·들어올리기·<strong>시차 베이스라인</strong></li>
</ul>

<ul>
<li><strong>TCP는 다른 네트워크도 됨</strong> — 막는 건 TCP가 아니라 <strong>NAT/방화벽</strong></li>
