2:T1637,<h1>DDS 완전 정리 (ROS2)</h1>
<h2>1. DDS란? — 개념</h2>
<p>ROS2는 여러 프로그램(노드)이 데이터를 주고받는다. 이 <strong>&quot;어떻게 주고받느냐&quot;</strong> 를 담당하는 통신 토대가 <strong>DDS(Data Distribution Service)</strong> 다.</p>
<p><strong>비유 — 단체 카톡방 📱</strong></p>
<ul>
<li>노드 = 사람들 / 토픽 = 채팅방 / DDS = 채팅 시스템 자체</li>
</ul>
<p><strong>ROS1 vs ROS2</strong></p>
<ul>
<li>ROS1: 중간에 <strong>총무(master)</strong> 가 연결 → 총무 죽으면 다 끊김 ❌</li>
<li>ROS2(DDS): 총무 없이 <strong>노드끼리 알아서 찾아 직접 대화</strong> ✅</li>
</ul>
<blockquote>
<p><strong>DDS = ROS2 노드들이 총무 없이 서로 찾아 직접 통신하게 해주는 표준 통신 방식.</strong></p>
</blockquote>
<h2>2. DDS의 3대 동작 원리</h2>
<table>
<thead>
<tr>
<th>원리</th>
<th>뜻</th>
</tr>
</thead>
<tbody><tr>
<td><strong>Pub/Sub (발행/구독)</strong></td>
<td>보내는 쪽(publisher)이 토픽에 던지면, 듣는 쪽(subscriber)이 받음</td>
</tr>
<tr>
<td><strong>Discovery (자동 탐색)</strong></td>
<td>총무 없이 노드끼리 &quot;나 여기 있어!&quot; 외쳐서 서로 발견</td>
</tr>
<tr>
<td><strong>QoS (품질 정책)</strong></td>
<td>토픽마다 &quot;확실히 전달 vs 빠르게 대충&quot; 등 전달 품질 설정</td>
</tr>
</tbody></table>
<hr>
<h2>3. ROS2 통신 3종류 — 전부 DDS Pub/Sub 위에서 돈다</h2>
<p>DDS가 원래 제공하는 건 <strong>오직 Pub/Sub(토픽)</strong>. Service·Action은 그걸 <strong>조합해 만든 상위 패턴</strong>이다.</p>
<table>
<thead>
<tr>
<th>방식</th>
<th>모양</th>
<th>비유</th>
<th>리비 예시</th>
</tr>
</thead>
<tbody><tr>
<td><strong>Topic</strong></td>
<td>계속 흘려보냄, 다대다</td>
<td>라디오 📻</td>
<td>카메라·라이다, <code>/cmd_vel</code></td>
</tr>
<tr>
<td><strong>Service</strong></td>
<td>요청→응답, 1:1</td>
<td>전화 ☎️</td>
<td>&quot;배터리 몇%?&quot; 묻고 답</td>
</tr>
<tr>
<td><strong>Action</strong></td>
<td>긴 작업+진행+취소</td>
<td>택배 📦</td>
<td>&quot;서가 가서 책 집기&quot;</td>
</tr>
</tbody></table>
<pre><code>Topic   = DDS Pub/Sub 그 자체
Service = 요청용 토픽 + 응답용 토픽
Action  = Service(목표·취소·결과) + Topic(피드백·상태)
</code></pre>
<blockquote>
<p>즉 <strong>모든 ROS2 통신은 밑바닥에서 DDS Pub/Sub로 굴러간다.</strong></p>
</blockquote>

<h2>4. DDS 종류 — Fast DDS vs Cyclone DDS</h2>
<p>DDS는 <strong>규격(약속)</strong> 이라 만든 회사가 여러 곳이다. 하는 일은 같고 <strong>만든 방식만 다르다</strong> (카톡 vs 라인).</p>
<h3>1️⃣ Fast DDS (eProsima) — 기본값</h3>
<ul>
<li><strong>원리</strong>: 표준 DDS를 풀스펙 구현 + 추가 기능. 같은 PC 노드끼린 <strong>공유메모리(Shared Memory)</strong> 로 직행 (네트워크 안 거침 → 빠름)</li>
<li><strong>기능</strong>: 공유메모리(대용량 초고속), Discovery Server(대규모 탐색 최적화), XML 세밀 설정</li>
<li><strong>언제</strong>: 그냥 시작 / 한 PC에서 영상·라이다 대용량 / 노드 수백 개 대규모</li>
</ul>
<h3>2️⃣ Cyclone DDS (Eclipse) — 가볍고 튼튼</h3>
<ul>
<li><strong>원리</strong>: 꼭 필요한 것만 단순 구현 → 가볍고 동작이 <strong>예측 가능, 잘 안 끊김</strong></li>
<li><strong>기능</strong>: 경량(저메모리), 안정적·결정적, 설정 간단(<code>CYCLONEDDS_URI</code>)</li>
<li><strong>언제</strong>: <strong>멀티로봇</strong> / 통신 꼬일 때 1순위 전환 / 라즈베리파이 등 저사양 / Nav2 불안정</li>
</ul>

<h2>한 줄 요약</h2>
<blockquote>
<p><strong>DDS</strong> = ROS2가 총무 없이 직접 통신하는 토대(Pub/Sub·Discovery·QoS).
<strong>통신 3종</strong>(Topic·Service·Action)도 전부 DDS Pub/Sub 위에서 동작.
<strong>종류</strong>는 Fast DDS(기본·기능많음)와 Cyclone DDS(가볍고 튼튼) — 멀티로봇/꼬일 때 Cyclone, 섞으면 안 됨.</p>
</blockquote>
