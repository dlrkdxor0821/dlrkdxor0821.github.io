2:T2ec0,<h1>OCR 학습 노트</h1>
<p>광학 문자 인식(OCR) 개념을 개발자 관점에서 정리. OCR이 뭔지, 실제 파이프라인 4단계, 인식 모델 내부 구조, 단순 텍스트를 넘어선 Document AI, 정확도 지표(CER/WER), 전처리의 역할과 &quot;딥러닝 OCR은 과한 전처리가 독&quot;이라는 반전, 클라우드 vs 온디바이스 선택까지.</p>
<hr>
<h2>OCR이 뭐냐</h2>
<p><strong>OCR(Optical Character Recognition, 광학 문자 인식)</strong> = 이미지 속 글자를 컴퓨터가 다룰 수 있는 <strong>텍스트 데이터</strong>로 바꾸는 기술.</p>
<p>예: 영수증 사진의 &quot;총액 12,000원&quot;이라는 <strong>글자 픽셀</strong> → <code>&quot;12000&quot;</code>이라는 <strong>문자열</strong>.</p>
<p>쓰임: 문서 스캔, 영수증·명함 자동 입력, 번호판 인식, 사진 속 외국어 번역, 공장 라벨·일련번호 읽기 등.</p>
<h2>실제 파이프라인은 4단계</h2>
<p>개념적으로는 &quot;검출 + 인식&quot; 2단계지만, 실무에선 전처리·후처리가 정확도를 좌우한다.</p>
<pre><code>① 전처리  →  ② 검출(Detection)  →  ③ 인식(Recognition)  →  ④ 후처리
</code></pre>
<table>
<thead>
<tr>
<th>단계</th>
<th>하는 일</th>
</tr>
</thead>
<tbody><tr>
<td>① 전처리</td>
<td>흑백화, 이진화, 기울기 보정, 노이즈 제거, 해상도 키우기</td>
</tr>
<tr>
<td>② 검출</td>
<td>글자가 <strong>어디 있는지</strong> 박스로 찾기 (DBNet, EAST 등)</td>
</tr>
<tr>
<td>③ 인식</td>
<td>박스 안 글자가 <strong>무슨 글자인지</strong> 읽기</td>
</tr>
<tr>
<td>④ 후처리</td>
<td>정규식·사전으로 교정, 신뢰도 필터링, 포맷 검증</td>
</tr>
</tbody></table>
<p>→ ②가 &quot;글자 위치 찾기&quot;(탐지), ③이 &quot;그 글자가 ㄱ인지 A인지&quot;(분류). 객체탐지와 비슷한 흐름.</p>
<h2>인식 모델 내부 구조</h2>
<table>
<thead>
<tr>
<th>방식</th>
<th>핵심</th>
<th>특징</th>
</tr>
</thead>
<tbody><tr>
<td><strong>CRNN + CTC</strong></td>
<td>CNN으로 특징 → RNN으로 순서 → CTC로 정렬</td>
<td>고전·표준, 가볍고 빠름</td>
</tr>
<tr>
<td><strong>Attention/Transformer</strong></td>
<td>문맥 보고 글자 예측</td>
<td>정확도 높음, 무거움</td>
</tr>
<tr>
<td><strong>TrOCR</strong></td>
<td>이미지→텍스트를 통째로</td>
<td>손글씨·어려운 글자에 강함</td>
</tr>
</tbody></table>
<blockquote>
<p><strong>CTC</strong>(Connectionist Temporal Classification): 글자 개수·위치를 미리 안 정해도 <strong>가변 길이 텍스트</strong>를 읽게 해주는 핵심 기법. OCR 공부하면 자주 나오는 용어.</p>
</blockquote>
<h2>OCR은 텍스트만 주는 게 아니다 — Document AI</h2>
<p>실무에서 진짜 중요한 건 <strong>글자 + 위치 + 구조</strong>다.</p>
<ul>
<li><strong>Bounding box</strong> — 각 글자/단어 좌표</li>
<li><strong>Confidence score</strong> — 인식이 얼마나 확실한지 (필터링 필수)</li>
<li><strong>Layout analysis</strong> — 제목/문단/표 구분</li>
<li><strong>Table extraction</strong> — 표를 행·열 구조로</li>
<li><strong>Key-Value extraction</strong> — &quot;이름: 홍길동&quot;에서 키-값 짝짓기 (서류 자동화의 핵심)</li>
</ul>
<p>→ 단순 글자 읽기를 넘어서면 이걸 <strong>Document AI / IDP(지능형 문서 처리)</strong> 라고 부른다.</p>
<h2>정확도 지표 (필수)</h2>

<th>의미</th>
</tr>
</thead>

<td><strong>CER</strong> (Character Error Rate)</td>
<td><strong>글자</strong> 단위 오류율 — OCR 표준</td>
</tr>
<tr>
<td><strong>WER</strong> (Word Error Rate)</td>

<blockquote>
<p>&quot;정확도 95%&quot;는 모호하다. <strong>CER/WER로 측정</strong>해야 비교가 된다 (낮을수록 좋음).</p>
</blockquote>
<h2>전처리 — 무엇을, 왜</h2>
<p>OCR 모델은 깨끗하고 또렷한 글자에 강하다. 전처리는 모델이 읽기 쉽게 이미지를 다듬는 작업.</p>
<table>
<thead>
<tr>
<th>단계</th>
<th>뭐 하나</th>
<th>왜</th>
</tr>
</thead>
<tbody><tr>
<td>Grayscale</td>
<td>컬러 → 흑백</td>
<td>색은 불필요, 연산↓</td>
</tr>
<tr>
<td>Resize/Upscale</td>
<td>작은 글자 키우기</td>
<td>작으면 못 읽음</td>
</tr>
<tr>
<td>Denoise</td>
<td>잡티 제거</td>
<td>점을 글자로 오인 방지</td>
</tr>
<tr>
<td>Contrast(CLAHE)</td>
<td>명암 키우기</td>
<td>흐릿한 글자 또렷하게</td>
</tr>
<tr>
<td><strong>Binarization(이진화)</strong></td>
<td>흑/백 2색으로</td>
<td>글자와 배경 분리 — 전처리 핵심</td>
</tr>
<tr>
<td><strong>Deskew</strong></td>
<td>기울기 보정</td>
<td>기울면 인식률 급락</td>
</tr>
<tr>
<td>Perspective 보정</td>
<td>비스듬한 걸 정면으로</td>
<td>영수증·간판</td>
</tr>
</tbody></table>
<p>이진화는 조명 균일하면 <strong>Otsu</strong>, 그림자·불균일하면 <strong>Adaptive threshold</strong>.</p>
<h2>⚠️ 반전: 요즘은 전처리를 덜 한다</h2>
<table>
<thead>
<tr>
<th></th>
<th>옛날 OCR (Tesseract)</th>
<th>요즘 딥러닝 OCR (PaddleOCR 등)</th>
</tr>
</thead>
<tbody><tr>
<td>전처리</td>
<td>많이 해야 잘 됨 (이진화 필수)</td>
<td>모델이 잡티·기울기를 어느 정도 알아서 처리</td>
</tr>
<tr>
<td>입력</td>
<td>흑백·이진화 선호</td>
<td><strong>컬러 원본 그대로</strong>가 더 나을 때 많음</td>
</tr>
</tbody></table>
<p>→ 딥러닝 OCR에 과하게 이진화하면 오히려 정보가 날아가 정확도가 떨어질 수 있다.</p>
<p><strong>실무 원칙:</strong></p>
<ol>
<li>일단 <strong>원본 그대로 돌려본다</strong></li>
<li>안 되면 <strong>뭐가 문제인지 진단</strong> (어두움? 기울었나? 작나?)</li>
<li><strong>그 문제에 맞는 전처리만</strong> 적용 (다 하지 말고)</li>
<li><strong>CER로 전/후 비교</strong> — 추측 말고 숫자로</li>
</ol>
<blockquote>
<p>premature optimization은 만악의 근원. 전처리도 <strong>문제를 본 다음에</strong> 최적화.</p>
</blockquote>

<h2>핵심 요약</h2>
<blockquote>
<p>OCR = 이미지 속 글자 → 텍스트. 실무는 전처리·검출·인식·후처리 4단계.
글자뿐 아니라 <strong>위치·구조·신뢰도</strong>까지 다루면 Document AI.
정확도는 <strong>CER/WER</strong>로 측정. 딥러닝 OCR은 <strong>일단 원본으로 해보고</strong> 안 될 때 문제에 맞는 전처리만 추가.</p>
