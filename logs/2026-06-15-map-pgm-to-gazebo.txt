2:T1ca1,<h1>map.pgm_To_Gazebo</h1>
<p>ROS2의 점유격자 맵(<code>map.pgm</code>)을 Gazebo 시뮬레이션 환경(<code>world.sdf</code>)으로 변환하는 원리를 정리한다.
실제 구현은 <code>make_candidate_world.py</code> 스크립트(PGM → self-contained Gazebo world 변환기)를 기준으로 한다.</p>
<p>핵심 아이디어는 한 줄로 요약된다.</p>
<blockquote>
<p><strong>PGM 점유 픽셀 → 가로 run으로 묶고 → 세로로 병합해 직사각형으로 분해 → 각 직사각형을 일정 높이 박스로 extrude → world.sdf</strong></p>
</blockquote>
<p>벡터 트레이스가 아니라 <strong>픽셀(래스터)을 직사각형으로 근사</strong>하는 방식이라, 비스듬한 벽은 작은 박스들이 계단처럼 쌓인 형태가 된다. (사람이 직선 벡터로 그리는 traffic_editor / rmf_site와 대비됨)</p>
<hr>
<h2>0. 입력 — PGM 점유격자</h2>
<p>ROS <code>map_server</code>가 저장하는 <strong>P5 (binary) PGM</strong>. 픽셀의 밝기로 점유 상태를 표현한다.</p>
<ul>
<li>검정(어두움) = 벽 / 점유(occupied)</li>
<li>흰색(밝음) = 빈 공간(free)</li>
</ul>
<p>함께 있는 <code>map.yaml</code>에서 변환에 필요한 메타데이터를 읽는다.</p>

<h2>1. 점유 픽셀만 골라내기 (이진화)</h2>
<p>밝기가 임계값보다 어두운 픽셀을 <strong>벽(점유)</strong> 으로 보고 boolean 마스크를 만든다.</p>
<pre><code class="language-python">occ_mask = img &lt; int(free_thresh * 255)
</code></pre>
<p>여기까지는 단순한 흑백 마스크다.</p>
<hr>
<h2>2. 행마다 가로 덩어리(run) 찾기</h2>
<p>각 행(row)을 훑으며 <strong>연속으로 점유된 구간</strong>을 <code>(시작열, 끝열)</code>로 묶는다.</p>
<pre><code class="language-python">def runs_in_row(row):
    # 한 행에서 True가 연속된 구간을 (c0, c) 튜플로 수집
    runs = set()
    c = 0
    while c &lt; len(row):
        if row[c]:
            c0 = c
            while c &lt; len(row) and row[c]:
                c += 1
            runs.add((c0, c))
        else:
            c += 1
    return runs
</code></pre>
<p>예: 한 행이 <code>..████..██..</code> 이면 → <code>(2,6)</code>, <code>(8,10)</code> 두 개의 run.</p>
<hr>
<h2>3. 세로로 병합 → 직사각형 분해 (핵심 알고리즘)</h2>
<p>위·아래 행에서 <strong>같은 <code>(시작열, 끝열)</code> run이 이어지면 하나의 직사각형으로 합친다.</strong></p>
<pre><code>행0:  ████      ← run (0,4) 열림
행1:  ████      ← 같은 run, 계속 진행
행2:  ████      ← 같은 run
행3:  ..        ← run 닫힘 → 사각형 (행0~행3, 열0~4) 확정
</code></pre>
<ul>
<li>위 행에 있던 run이 다음 행에 없으면 → 사각형을 <strong>닫아서 저장</strong> <code>(r0, c0, r1, c1)</code></li>
<li>새 run이 나타나면 → 새 사각형 <strong>열기</strong></li>
</ul>
<pre><code class="language-python">def decompose_to_rects(occ_mask):
    rects = []
    open_rect = {}                 # (c0, c1) -&gt; 시작 행 r0
    for r in range(occ_mask.shape[0]):
        cur = runs_in_row(occ_mask[r])
        for k in [k for k in open_rect if k not in cur]:  # 끝난 run 닫기
            rects.append((open_rect[k], k[0], r, k[1]))
            del open_rect[k]
        for run in cur:            # 새 run 열기
            if run not in open_rect:
                open_rect[run] = r
    for run, r0 in open_rect.items():  # 마지막 행까지 열린 run 마감
        rects.append((r0, run[0], occ_mask.shape[0], run[1]))
    return rects
</code></pre>
<p>결과: 벽 픽셀 전체가 <strong>겹치지 않는 직사각형 리스트</strong>로 분해된다.</p>
<hr>
<h2>4. 픽셀 좌표 → 월드 좌표 변환</h2>
<p>직사각형의 픽셀 위치를 실제 미터 좌표(중심 <code>cx,cy</code> + 크기 <code>sx,sy</code>)로 바꾼다.</p>
<pre><code class="language-python">cx = (c0 + c1) / 2 * res + ox       # 중심 x
cy = (H - (r0 + r1) / 2) * res + oy  # 중심 y  ← 이미지 row 반전
sx = (c1 - c0) * res                 # 가로 크기
sy = (r1 - r0) * res                 # 세로 크기
</code></pre>
<p>주의할 점 두 가지:</p>
<ul>
<li><code>resolution</code>을 곱해 <strong>픽셀 → 미터</strong>, <code>origin</code>을 더해 <strong>맵 원점 보정</strong></li>
<li>이미지는 위쪽이 0행이지만 Gazebo y축은 위로 증가 → <strong><code>H - row</code>로 위아래를 뒤집어야</strong> 좌표가 맞는다</li>
</ul>
<hr>
<h2>5. 박스로 &quot;위로 세우기&quot; (extrude)</h2>
<p>각 직사각형을 일정 높이(예: <code>WALL_HEIGHT = 0.72m</code>) 박스로 만들어 SDF에 기록한다. 2D 평면 사각형을 z축으로 끌어올리는(extrude) 단계다.</p>
<pre><code class="language-xml">&lt;collision name=&quot;wall_0000&quot;&gt;
  &lt;pose&gt;{cx} {cy} 0.36 0 0 0&lt;/pose&gt;            &lt;!-- z = 높이/2 --&gt;
  &lt;geometry&gt;&lt;box&gt;&lt;size&gt;{sx} {sy} 0.72&lt;/size&gt;&lt;/box&gt;&lt;/geometry&gt;
&lt;/collision&gt;
&lt;visual name=&quot;wall_0000_v&quot;&gt;
  &lt;pose&gt;{cx} {cy} 0.36 0 0 0&lt;/pose&gt;
  &lt;geometry&gt;&lt;box&gt;&lt;size&gt;{sx} {sy} 0.72&lt;/size&gt;&lt;/box&gt;&lt;/geometry&gt;
&lt;/visual&gt;
</code></pre>
<p>여기에 바닥(<code>ground_plane</code>), 조명(<code>sun</code>), 물리/센서 플러그인까지 인라인으로 붙여 <strong>self-contained <code>world.sdf</code></strong> 를 완성한다. <code>model://</code> 외부 리소스 경로가 필요 없어 그대로 띄울 수 있다.</p>
