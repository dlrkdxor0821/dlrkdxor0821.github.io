2:T168f,<h1>NCNN 학습 노트</h1>
<p>온디바이스 추론 엔진 NCNN 개념 정리. AI 모델의 학습/추론 단계 구분, NCNN의 위치(ONNX·TFLite와 같은 급의 대체 런타임), 빠르고 가벼운 이유, 모델 파일 구조, 실제 배포 흐름, YOLO 예시, 한계까지.</p>
<hr>
<h2>먼저 큰 그림부터</h2>
<p>AI 모델은 보통 두 단계를 거친다.</p>
<ol>
<li><strong>학습(Training)</strong> — 데이터로 모델을 똑똑하게 만드는 과정. 무겁고 GPU 필요. → <strong>PyTorch</strong> 영역</li>
<li><strong>추론(Inference)</strong> — 다 만든 모델로 실제 예측하는 과정. → <strong>배포 단계</strong></li>
</ol>
<p>문제는, 학습은 좋은 PC에서 하지만 <strong>실제 서비스는 폰이나 라즈베리파이처럼 힘 약한 기기</strong>에서 돌아가야 할 때가 많다는 것. PyTorch를 그대로 폰에 올리면 너무 무겁고 느리다.</p>
<p>그래서 <strong>추론만 가볍게 전담하는 도구</strong>가 따로 필요한데, 그게 NCNN이다.</p>
<h2>NCNN이 정확히 뭐냐</h2>
<blockquote>
<p>Tencent가 만든 <strong>C++ 기반 추론 전용 엔진.</strong> 처음부터 목표가 딱 하나 — <strong>&quot;ARM(폰·라즈베리파이) 같은 작은 기기에서 모델을 최대한 빠르고 가볍게 돌린다.&quot;</strong></p>
</blockquote>
<p>핵심은 <strong>추론만</strong> 한다는 것. 학습 기능은 아예 없다. &quot;다 만든 모델을 실행하는 플레이어&quot;라고 보면 된다.</p>
<h2>어디에 위치하냐 (이게 중요)</h2>
<p>NCNN은 TFLite나 ONNX Runtime을 <strong>빠르게 해주는 게 아니라, 그것들과 같은 자리에서 경쟁하는 대체재</strong>다.</p>
<pre><code>PyTorch 로 학습
   ↓ export
ONNX (프레임워크 간 공용 교환 포맷)
   ↓ 여기서 배포 런타임 선택
   ├─ ONNX Runtime   (범용, 서버·데스크톱)
   ├─ TFLite         (구글·안드로이드 생태계)
   └─ NCNN           (ARM·모바일 극한 최적화)   ← 같은 급
</code></pre>
<p>즉 &quot;ONNX/TFLite <strong>위</strong>에 NCNN&quot;이 아니라 &quot;ONNX/TFLite <strong>대신</strong> NCNN&quot;이다.</p>
<h2>왜 빠르고 가벼운가 (NCNN의 자랑 3가지)</h2>
<ul>
<li><strong>의존성이 없다</strong> — BLAS, protobuf 같은 무거운 외부 라이브러리를 안 쓴다. 그래서 빌드 결과물이 작고 어디든 잘 들어간다</li>
<li><strong>ARM에 손최적화</strong> — ARM 칩의 가속 명령(NEON)에 맞춰 연산을 직접 깎아서 CPU만으로도 빠르다</li>
<li><strong>GPU도 지원</strong> — Vulkan으로 폰 GPU나 라즈베리파이 GPU 가속도 가능 (CUDA 없이도)</li>
</ul>
<h2>모델은 어떻게 생겼나</h2>
<p>NCNN 모델은 항상 <strong>파일 2개 한 쌍</strong>이다.</p>
<table>
<thead>
<tr>
<th>파일</th>
<th>내용</th>
</tr>
</thead>
<tbody><tr>
<td><code>.param</code></td>
<td>네트워크 <strong>구조</strong> (어떤 레이어가 어떻게 연결됐는지)</td>
</tr>
<tr>
<td><code>.bin</code></td>
<td><strong>가중치</strong> (학습으로 얻은 숫자값)</td>
</tr>
</tbody></table>
<p>다른 프레임워크 모델을 이 두 파일로 <strong>변환</strong>해서 사용한다.</p>
<h2>실제 흐름 한 번에</h2>
<pre><code>PyTorch 학습
  → ONNX 로 export
  → NCNN 포맷(.param/.bin)으로 변환
  → (선택) 양자화로 더 작고 빠르게  ※ 정밀도 살짝 낮춰 속도↑
  → 기기에 올려서 추론 실행
</code></pre>
<h2>대표 예시</h2>
<p>PC(GPU)에서 학습한 <strong>YOLO</strong> 객체탐지 모델을 NCNN으로 변환 → <strong>라즈베리파이에서 실시간으로 카메라 영상 속 물체를 탐지</strong>. GPU 없는 작은 기기에서 YOLO 돌릴 때 정말 흔한 조합이다.</p>
<h2>한계도 솔직히</h2>
<ul>
<li><strong>추론 전용</strong> (학습 불가)</li>
<li>변환할 때 <strong>NCNN이 지원 안 하는 연산</strong>이 있으면 직접 손봐야 함</li>
<li>문서가 영어/중국어 위주라 처음엔 좀 헤맬 수 있음</li>
</ul>
<hr>
<h2>핵심 요약</h2>
<blockquote>
<p>PyTorch(연구) → ONNX(교환) → <strong>NCNN(ARM 기기에서 빠르게 실행)</strong>. 작은 기기용 경량 추론 엔진.</p>
</blockquote>
<hr>
