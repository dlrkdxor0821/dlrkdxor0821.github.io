🤖 리비(Libby) 다중로봇 시스템 — 종합 정리

설계 논의를 한 곳에 정리. 크게 ① 시스템 구조 ② 인식·파지 ③ 하드웨어 ④ 통신 ⑤ 성능 으로 묶었다.

1. 시스템 아키텍처 (핵심 결론)

원칙: UI 3개, 두뇌 1개. 모든 UI는 백엔드를 거치고, 로봇을 직접 건드리지 않는다.

관리자 UI ──┐
사용자 UI ──┼──▶ [백엔드: 디스패처 + 상태/큐 + DB] ──▶ 로봇 A/B/C
로봇 UI  ──┘                                        (상태 발행 ▲)

UI 구성 + 기술스택

UI 역할 입력/출력 기술
관리자 관제·제어·선점 (권한 최상) 입출력 PyQt (rclpy 직접 연동)
사용자 요청·피드백 입력 웹 (앱 X, QR로 바로)
로봇(LCD) 상태 표시 출력 전용 Flask + 키오스크 브라우저

핵심 설계 결정

  • 음성이 사용자 주 입력 → 비터치 LCD 문제 해결 (폰 웹은 보조)
  • 권한 계층: 관리자 > 사용자 (관리자는 선점·비상정지 가능, 잠금 예외)
  • BUSY는 내부 상태 — 사용자에겐 "잠금"이 아니라 "가는 중 / 대기 N번째 / 지금 불가" 로 보임
  • QR은 robot_id를 운반하는 껍데기일 뿐 (판별·트리거의 주체가 아님). NFC·번호·버튼으로 교체 가능

요청 라우팅 — 두 갈래

유형 예 처리
대면 요청 추천·위치안내 앞에 있는 그 로봇이 처리
작업 요청 도서 이송 디스패처가 최적 로봇 배정

디스패치 (SR-DSP — 현재 요구사항 문서에서 빠진 부분 ⚠️)

  • 배정 기준 = 실제 경로 ETA (직선거리 ❌, Nav2로 계산 ✅)
  • 다 바쁘면 큐, 없으면 "지금 불가"
  • RMF(경매/입찰 방식)는 표준이지만 설정이 무거움 → 3대 규모면 직접 cost 함수 디스패처 추천
def assign_robot(task):
    candidates = [r for r in robots if r.state == 'IDLE']
    if not candidates:
        task_queue.add(task)            # 다 바쁘면 큐
        return
    best = min(candidates, key=lambda r: cost(r, task))   # 비용 최소
    best.assign(task)

def cost(robot, task):
    eta = path_eta(robot.pose, task.location)   # 실제 경로 예상시간
    eta += battery_penalty(robot)
    eta -= priority_bonus(task)
    return eta

⚠️ 웹↔ROS2 다리 필요

  • PyQt → rclpy 직접 연결 ✅
  • 웹(사용자/로봇) → ROS2 DDS에 직접 못 붙음 → rosbridge 또는 FastAPI 게이트웨이 경유

2. 인식·파지 (모방학습 없이 기구학으로)

YOLO OBB(각도) → 깊이 측정 → Grasp Pose → 역기구학(IK) → movel 실행

깊이 얻는 법 (뎁스캠 없이도 가능)

방법 적합
평면 가정 + 호모그래피 테이블 위 픽앤플레이스 (최고)
물체 실제크기 역산 OBB 픽셀크기로 Z = fx·W/w 계산
Eye-in-hand 모션시차 카메라가 팔에 부착 (로봇이 베이스라인 정확히 앎, 가변 가능)
AprilTag / 스테레오 마커 가능 / 카메라 2대

Eye-in-hand 모션시차 핵심: 팔을 순수 평행이동(movel) → Z = fx·B/d. 깊이 오차는 Z²/(fx·B) 에 비례 → 먼 물체일수록 많이 움직여라(B↑). 대응점은 cv2.triangulatePoints, hand-eye 캘리브 필수.

로봇 모션 명령

  • movej = 관절 보간 (빠름, TCP 경로 곡선) → 자유공간 이동
  • movel = TCP 직선 보간 (정밀) → 접근·들어올리기·시차 베이스라인

4. 통신·네트워크

원격 카메라 보기

방법 용도
웹 스트리밍(Flask MJPEG) 빨리 보기 (제일 쉬움)
ROS2 토픽 + Foxglove 시스템 통합
소켓 직접 학습·완전제어

다른 네트워크 연결

  • TCP는 다른 네트워크도 됨 — 막는 건 TCP가 아니라 NAT/방화벽
  • Tailscale(VPN) 으로 묶으면 TCP·ROS2 둘 다 "같은 LAN처럼" 동작
  • 주의: ROS2 DDS는 멀티캐스트라 같은 LAN 전제 (오히려 TCP보다 까다로움)
