2:T1b3b,<h1>Odometry 정리 — 시작점 기준 상대 위치 추정</h1>
<blockquote>
<p><strong>Odometry는 시작점을 기준으로 상대적인 위치를 측정하는 것.</strong>
(출처: SMG0317-Odometry 교육자료 + ROS2 개발자 보강)
흐름: <strong>정의 → 모션 모델(rot–trans–rot) → 노이즈/드리프트 → ROS odom frame &amp; TF → 보강 개념</strong></p>
</blockquote>
<hr>
<h2>Part 0. 한 줄 정의</h2>
<p>모터(바퀴)에 달린 <strong>엔코더의 회전수</strong>와 <strong>IMU(관성 측정 장비)의 기울기</strong> 등을 측정하여, <strong>시작점(0,0)을 기준으로 지금 얼마나 움직였는지</strong> 상대 위치를 추정하는 방법.</p>
<blockquote>
<p>= <strong>추측 항법(Dead Reckoning)</strong>: <em>&quot;(0,0)에서 초당 1m로 3초 직진했으니 지금 (3,0)일 거야&quot;</em> 라고 <strong>계산으로</strong> 위치를 추정.</p>
</blockquote>
<hr>
<h2>Part 1. 오도메트리 모션 모델 (rotation → translation → rotation)</h2>
<p>상대 오도메트리를 추출할 때, 두 시점 사이의 움직임을 <strong>세 단계</strong>로 분해한다 (Probabilistic Robotics, Fig 5.7):</p>
<pre><code>        δ_rot2
         ◜
    δ_trans
   ◜
 ●  δ_rot1
 (이전 pose)        →        (현재 pose)
</code></pre>
<table>
<thead>
<tr>
<th>단계</th>
<th>의미</th>
</tr>
</thead>
<tbody><tr>
<td><strong>δ_rot1</strong></td>
<td>먼저 목표 방향으로 <strong>회전</strong></td>
</tr>
<tr>
<td><strong>δ_trans</strong></td>
<td>그 방향으로 <strong>직선 이동</strong></td>
</tr>
<tr>
<td><strong>δ_rot2</strong></td>
<td>도착 후 최종 자세로 <strong>또 한 번 회전</strong></td>
</tr>
</tbody></table>
<p>수식 (motion_model_odometry):</p>
<pre><code>δ_rot1  = atan2(ȳ&#39;−ȳ, x̄&#39;−x̄) − θ̄
δ_trans = √((x̄−x̄&#39;)² + (ȳ−ȳ&#39;)²)
δ_rot2  = θ̄&#39; − θ̄ − δ_rot1
</code></pre>
<p>→ 이 세 값이 <strong>상대 운동의 충분 통계량</strong>. 알고리즘은 <code>p(xₜ | uₜ, xₜ₋₁)</code> (오도메트리로부터 현재 pose의 확률)을 계산한다.</p>
<blockquote>
<p>핵심: 회전·직선·회전 세 파라미터는 <strong>독립적인 노이즈</strong>의 영향을 받는다고 가정 → 그래서 확률적으로 다룬다. ([[2026-06-16-amcl]]의 motion update가 이걸 씀)</p>
</blockquote>
<hr>
<h2>Part 2. 노이즈와 드리프트 — 오도메트리의 숙명</h2>
<p>바퀴는 <strong>미끄러지고(slip)</strong>, 바닥은 <strong>울퉁불퉁</strong>하다. 그래서 엔코더가 &quot;1m 갔다&quot;고 해도 실제론 0.95m일 수 있다.</p>
<ul>
<li>이 오차는 <strong>계속 누적(accumulate)</strong> 된다 → 시간이 갈수록 추정 위치가 실제와 벌어짐 = <strong>드리프트(drift)</strong></li>
<li>그래서 오도메트리만으로는 <strong>장시간/장거리 정확한 절대 위치를 알 수 없다.</strong></li>
<li>보정하려면 외부 기준(지도+LiDAR)이 필요 → [[2026-06-16-amcl]] (AMCL이 <code>map→odom</code>으로 이 드리프트를 잡아줌)</li>
</ul>
<blockquote>
<p>오도메트리 = <strong>단기적으론 부드럽고 정확, 장기적으론 드리프트.</strong> 이 성질이 아래 TF 구조의 이유가 된다.</p>
</blockquote>
<hr>
<h2>Part 3. ROS에서의 Odometry — odom frame</h2>
<p>Odometry는 ROS에서 <strong><code>odom</code> frame</strong>으로 구현되며, 엔코더 또는 IMU로 위치를 추정한다. (엔코더·IMU를 못 쓸 경우 <strong>Visual Odometry</strong> — 카메라 관측으로 odom 추정 가능)</p>
<h3>TF 프레임 구조</h3>
<pre><code>earth  →  map  →  odom  →  base_link
</code></pre>
<blockquote>
<p><code>map</code>은 여러 개의 <code>odom</code>을 가질 수 있으나, <code>odom</code>은 단 하나의 <code>map</code>을 가진다.</p>
</blockquote>
<table>
<thead>
<tr>
<th>frame</th>
<th>정의</th>
<th>특징</th>
</tr>
</thead>
<tbody><tr>
<td><strong>earth</strong></td>
<td>여러 로봇이 서로 다른 map 위에 존재하도록 묶는 최상위</td>
<td>멀티로봇/멀티빌딩용</td>
</tr>
<tr>
<td><strong>map</strong></td>
<td>world fixed frame, earth에 고정된 전역 기준</td>
<td><strong>불연속</strong>(AMCL 보정 시 순간이동처럼 점프 가능) but 장기 global reference로 유용</td>
</tr>
<tr>
<td><strong>odom</strong></td>
<td>world fixed frame, 연속적인 로컬 기준</td>
<td><strong>연속적</strong>(부드러움) but 드리프트로 장기 기준엔 부적합</td>
</tr>
<tr>
<td><strong>base_link</strong></td>
<td>로봇 자체의 위치를 나타내는 frame</td>
<td>odom/map을 통해 위치 추정</td>
</tr>
</tbody></table>
<blockquote>
<p>왜 <code>map</code>과 <code>odom</code>을 분리하나? → <strong>map은 정확하지만 점프(불연속), odom은 부드럽지만 드리프트.</strong> 둘을 분리해 &quot;부드러움(odom)&quot;과 &quot;정확함(map)&quot;을 모두 얻는다. 자세한 TF 메커니즘은 [[2026-06-16-tf]] 참고.</p>
</blockquote>
<hr>
<h2>Part 4. ROS2 개발자 보강 — 실무에서 알아야 할 것</h2>
<ul>
<li><strong>표준 메시지</strong>: <code>nav_msgs/Odometry</code> (pose + twist + covariance). 보통 <code>/odom</code> 토픽으로 발행.</li>
<li><strong>누가 odom→base_link TF를 쏘나</strong>: 바퀴 컨트롤러(<code>ros2_control</code> diff_drive_controller) 또는 가제보 플러그인, 혹은 <code>robot_localization</code> 노드.</li>
<li><strong>센서 융합</strong>: <code>robot_localization</code> 패키지의 <strong>EKF/UKF</strong> 노드가 휠 오도메트리 + IMU + (실외) GPS를 융합해 더 안정적인 odom을 만든다. → 칼만 필터의 실전 용례 ([[2026-06-16-filter]] 참고)</li>
<li><strong>REP-105</strong>: ROS 좌표계 표준(<code>map</code>/<code>odom</code>/<code>base_link</code> 의미와 관계)을 정의. 프레임 이름을 임의로 바꾸지 말 것.</li>
<li><strong>odom은 navigation의 입력</strong>: Nav2의 Controller·Velocity Smoother가 <code>odom</code>을 받아 제어 ([[2026-06-16-nav2-pkg]]).</li>
</ul>
<hr>
<h2>한 줄 요약</h2>
<blockquote>
<p><strong>Odometry는 엔코더·IMU로 &quot;시작점 대비 얼마나 움직였나&quot;를 계산하는 추측항법이다.</strong> 회전–직선–회전 모델로 분해되고 노이즈가 누적돼 <strong>드리프트</strong>가 생기므로, 단기엔 정확하지만 장기엔 부정확하다. 그래서 ROS는 **연속적인 <code>odom</code>**과 <strong>정확한 <code>map</code></strong> 프레임을 분리하고, AMCL/EKF가 그 사이 드리프트를 보정한다.</p>
</blockquote>
