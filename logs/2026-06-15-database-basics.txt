<h2>3. 접근 방식 — 코드가 DB에 어떻게 말 거느냐</h2>
<h3>축 ① SQL 직접 쓰기 vs ORM (통역사)</h3>
<table>
<thead>
<tr>
<th>방식</th>
<th>한마디</th>
<th>언제 주로 쓰나</th>
</tr>
</thead>
<tbody><tr>
<td><strong>생 SQL</strong></td>
<td>SQL 직접 작성</td>
<td>복잡 통계·집계, 성능 극한 튜닝, 단순 스크립트</td>
</tr>
<tr>
<td><strong>ORM</strong></td>
<td>파이썬 코드→SQL 자동번역</td>
<td>모델 많은 일반 앱, 팀 협업·장기 유지보수, 반복 CRUD</td>
</tr>
</tbody></table>
<ul>
<li><strong>생 SQL</strong>: <code>cursor.execute(&quot;SELECT * FROM books WHERE id = 1&quot;)</code> → 정확·빠름, SQL 다 써야 함</li>
<li><strong>ORM</strong>: <code>session.query(Book).filter(Book.id == 1)</code> → 깔끔·편함, 중간 번역 계층</li>
<li>비유: 생 SQL = 현지어로 직접 대화 / ORM = 통역사 두고 대화</li>
</ul>
<h3>축 ② 동기 vs 비동기 (기다리는 방식)</h3>
<table>
<thead>
<tr>
<th>방식</th>
<th>한마디</th>
<th>언제 주로 쓰나</th>
</tr>
</thead>
<tbody><tr>
<td><strong>동기(sync)</strong></td>
<td>답 올 때까지 줄 서서 대기</td>
<td>단순한 앱, 순차 처리, 배치·스크립트</td>
</tr>
<tr>
<td><strong>비동기(async)</strong></td>
<td>기다리는 동안 다른 일 처리</td>
<td>동시 요청 많은 웹서버/API, 실시간</td>
</tr>
</tbody></table>
