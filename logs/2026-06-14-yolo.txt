<p>객체 탐지 모델 YOLO 개념 정리. 학습/저장/배포 흐름, 모델 종류, 동작 원리, 학습·추론 옵션, 결과 그래프 해석, 핵심 요약까지.</p>

<h2>1. YOLO란? — &quot;학습시킨다&quot;의 의미</h2>
<p>YOLO(You Only Look Once)는 이미지를 보고 한 번에 다음을 예측하는 <strong>객체 탐지(Object Detection)</strong> 모델이다.</p>
<ol>
<li><strong>어디에</strong> 물체가 있는지 — Bounding Box (박스 좌표)</li>
<li><strong>무엇인지</strong> — 클래스 (사람/자동차/책 …)</li>
<li><strong>얼마나 확신하는지</strong> — Confidence</li>
</ol>
<p><strong>&quot;학습시킨다(train)&quot;</strong> = 모델 내부의 수백만 개 가중치(weight)를 데이터로부터 조정해, 예측이 정답에 가까워지도록 최적화하는 과정.</p>
<pre><code>1. 이미지 입력  →  2. 모델이 박스/클래스 예측  →  3. 정답(라벨)과 비교
                                                        ↓
6. 가중치 업데이트  ←  5. 오차를 거꾸로 전파  ←  4. 오차(Loss) 계산
        ↑                                              
        └──────────── 이걸 수천~수만 번 반복 ────────────┘
</code></pre>
<p><strong>학습에 필요한 것</strong>: 데이터셋(이미지 + 라벨), 라벨링(정답 박스/클래스 표시), GPU, 하이퍼파라미터(epoch·learning rate·batch 등).</p>
<blockquote>
<p>비유: 데이터셋=문제집+정답지, 학습=풀고 채점하고 틀린 것 고치기 반복, 학습된 모델=새 문제도 풀 수 있는 상태.</p>
</blockquote>

<h2>3. 학습된 모델은 어디에 저장되나?</h2>
<p>학습이 끝나면 모델은 <strong>하나의 파일</strong>로 디스크에 저장된다. Ultralytics YOLO 기준:</p>
<pre><code>runs/detect/train/weights/
├── best.pt    ← 학습 중 성능이 가장 좋았던 가중치
└── last.pt    ← 마지막 epoch의 가중치
</code></pre>
<ul>
<li><code>.pt</code> = PyTorch 모델 파일 (가중치 숫자들의 묶음)</li>
<li><strong>모델 = 파일 하나(best.pt)</strong> 라고 생각하면 된다. USB로 복사하든 클라우드에 올리든 자유.</li>
</ul>
<h3>다른 로컬 환경에서 학습 가능? → 가능</h3>
<ul>
<li>내 노트북, 회사 데스크탑, Google Colab(무료 GPU), 클라우드 서버 등 어디서든 가능. 결과물은 똑같은 <code>.pt</code> 파일.</li>
<li>단, 학습은 연산량이 커서 GPU 없으면 매우 느리다 → GPU 환경 권장.</li>
</ul>

<h2>4. YOLO 동작 원리</h2>
<h3>4.1 그리드 분할 — 누가 어떤 물체를 맡을지 나누기</h3>
<p>입력 이미지를 SxS 그리드로 분할. 규칙: <strong>물체의 &quot;중심&quot;이 들어있는 칸이 그 물체를 예측할 책임</strong>을 진다.</p>
<pre><code>┌──┬──┬──┬──┬──┬──┬──┐
│  │  │🐶│  │  │  │  │  ← 강아지 중심이 있는 칸이 강아지 담당
├──┼──┼──┼──┼──┼──┼──┤
│  │  │  │  │  │🚗│  │  ← 자동차 중심이 있는 칸이 자동차 담당
└──┴──┴──┴──┴──┴──┴──┘
</code></pre>
<p>각 셀은 <strong>Bounding Box(x,y,w,h) + Confidence + Class 확률</strong> 을 예측. 모든 칸이 동시에 예측 → 한 번의 forward pass로 빠르게 처리.</p>
<h3>4.2 NMS(Non-Maximum Suppression) — 같은 물체의 중복 박스 제거</h3>
<p>한 물체에 여러 박스가 겹쳐 잡히면 가장 확실한 하나만 남긴다.</p>
<pre><code>[NMS 전] 강아지 1마리에 박스 3개      [NMS 후] 1개만 남김
   ┌────────┐                          ┌────────┐
   │┌───────┴┐                         │  🐶    │
   ││ 🐶     │                          └────────┘
   │└┬───────┘
   └─┴──────┘
</code></pre>
<p><strong>&quot;같은 물체&quot;인지 판단 기준 = 겹침 정도(IoU)</strong></p>
<ul>
<li>많이 겹침(IoU 높음) + 같은 클래스 → 같은 물체 → 점수 낮은 박스 제거</li>
<li>안 겹침(IoU 낮음) → 다른 물체 → 둘 다 유지</li>
</ul>
<blockquote>
<p><strong>핵심: NMS는 겹침이 큰 박스끼리만 경쟁시켜 하나를 남긴다. 다른 물체(겹침 적음)는 건드리지 않는다.</strong></p>
</blockquote>
<h3>4.3 1-stage vs 2-stage — 왜 빠른가</h3>
<pre><code>2-stage (R-CNN 계열): 후보 영역 찾기 → 각각 분류   (정확하지만 느림)
1-stage (YOLO):        한 번에 위치+분류 동시 예측   (빠르고 실시간)
</code></pre>
<p>YOLO는 탐지를 <strong>단일 신경망의 회귀 문제</strong>로 봐서 영역 제안 단계 없이 한 번에 끝낸다. 전체 이미지를 한 번에 보므로 맥락(context) 파악 → 배경 오탐이 적다. (초기엔 작은·밀집 물체에 약했으나 v3~v11에서 개선)</p>
<hr>
<h2>5. 학습 설정 — Epoch / Batch / Iteration</h2>
<p><strong>Epoch(에폭)</strong> = 전체 학습 데이터셋을 처음부터 끝까지 <strong>한 번 다 학습한 횟수</strong>.</p>
<blockquote>
<p>문제집 한 권 = 전체 데이터셋. 1 epoch = 문제집 1번 다 풀기. 100 epochs = 100번 반복. 여러 번 반복해야 패턴을 제대로 학습한다.</p>
</blockquote>
<table>
<thead>
<tr>
<th>용어</th>
<th>뜻</th>
</tr>
</thead>
<tbody><tr>
<td><strong>Epoch</strong></td>
<td>전체 데이터를 1번 다 본 것</td>
</tr>
<tr>
<td><strong>Batch</strong></td>
<td>한 번에 다 못 보니 잘게 나눈 묶음 (예: 16장씩)</td>
</tr>
<tr>
<td><strong>Iteration</strong></td>
<td>배치 1개를 학습하는 1번의 단계</td>
</tr>
</tbody></table>
<p>예) 데이터 1,600장, batch=16 → <strong>1 epoch = 100 iterations</strong> (1600 ÷ 16)</p>
<p><strong>몇으로 설정?</strong></p>
<ul>
<li>너무 적으면(예: 5) → 덜 배움 = <strong>과소적합(underfitting)</strong></li>
<li>너무 많으면(예: 1000) → 통째로 외워버려 새 데이터에 약함 = <strong>과적합(overfitting)</strong></li>
<li>YOLO 기본값은 보통 <code>epochs=100</code>. loss/지표 그래프를 보며 더 이상 안 좋아지면 조절/중단.</li>
</ul>
<pre><code class="language-bash">yolo train model=yolo11n.pt data=data.yaml epochs=100 batch=16 imgsz=640
</code></pre>

<h3>30초 버전</h3>
<blockquote>
<p>&quot;YOLO는 You Only Look Once의 약자로, 이미지를 한 번만 보고 객체의 위치와 종류를 동시에 예측하는 객체 탐지 모델입니다. 기존 방식이 &#39;후보 영역 찾기 → 분류&#39;를 따로 했다면, YOLO는 이를 하나의 회귀 문제로 통합해 한 번에 처리하므로 매우 빠르고 실시간 처리가 가능합니다.&quot;</p>
</blockquote>
<h3>외울 핵심 흐름 5개</h3>
<p><strong>① You Only Look Once → ② 1-stage(한 번에) → ③ 그리드 분할 → ④ 박스+Confidence+Class 동시 예측 → ⑤ NMS로 중복 제거</strong></p>
