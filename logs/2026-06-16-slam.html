SLAM 정리 — 위치추정과 지도작성을 동시에

SLAM이란 동시적으로 위치를 추정하고 지도를 작성하는 기술이다. (Simultaneous Localization and Mapping / 출처: SMG0317-SLAM 교육자료 + ROS2 개발자 보강) 흐름: 정의 → 닭과 달걀 문제 → 작동 6단계 → 종류 → 3대 난제 → Localization과의 관계

Part 0. 한 줄 정의

로봇이 주어진 공간에서 주변 환경을 인식하고 스스로 위치를 파악함과 동시에 지도를 만들어가는 기술. 현재 어디 있는지 정확히 파악하며, 이동하면서 주변 환경의 지도를 구축하는 것.

로봇이 스스로 알아서 움직이려면 SLAM이 필요하다 (지도가 없는 미지의 공간에서).

Part 1. 왜 어려운가 — "닭과 달걀" 문제

정확한 지도를 만들려면  →  내 위치를 정확히 알아야 함
내 위치를 정확히 알려면  →  정확한 지도가 있어야 함

둘 다 모르는 상태에서 둘을 동시에 풀어야 한다. 이게 SLAM이 어려운 본질이다.

Part 2. SLAM 작동 원리 (6단계)

  1. 초기화: 시작 위치에서 센서 데이터를 수집하고 초기 지도를 생성.
  2. 데이터 획득: 이동하면서 센서 데이터를 지속적으로 수집.
  3. 데이터 연관 (Data Association): 현재 데이터와 이전 데이터를 비교해 특징점을 매칭하고 이동량을 계산.
  4. 지도 업데이트: 계산된 이동량을 기반으로 지도를 업데이트.
  5. 위치 추정: 업데이트된 지도와 센서 데이터로 현재 위치를 추정.
  6. 반복: 3~5단계를 반복하여 지도를 점차 완성하고 정확한 위치 추정을 수행.

3↔5가 서로를 보정하며 도는 게 SLAM의 핵심 루프. ([[2026-06-16-odometry]]가 이동량의 초기 추정을 제공)

Part 3. SLAM의 종류

종류 센서 차원 장점 단점
Lidar SLAM 레이저 센서 2D/3D 정확도 높음 비용 높음
Visual SLAM 카메라 (주로) 단안/스테레오 비용 저렴 조명 변화·텍스처에 성능 저하

ROS2 실무: 2D는 slam_toolbox(Lidar 기반)가 표준. Visual SLAM은 ORB-SLAM3, RTAB-Map 등.

Part 4. SLAM의 3대 난제와 해결 방향

난제 내용 해결
데이터 연관 문제 특징점 추출·매칭 과정의 오류가 지도 정확도에 영향 강건한 특징점, 이상치 제거(RANSAC)
루프 클로저 (Loop Closure) 처음 방문했던 장소로 다시 돌아왔을 때, 누적된 드리프트로 이전 지도와 불일치 → 이를 해소 "여기 와봤다"를 인식해 그래프 전체를 보정
실시간 처리 많은 센서 데이터를 실시간 처리 → 높은 연산 능력 필요 front-end(추적)/back-end(최적화) 분리

루프 클로저가 SLAM의 꽃: 한 바퀴 돌아 출발점을 다시 보면, 그동안 쌓인 오도메트리 드리프트를 한 번에 펴서 지도를 닫는다. (Graph SLAM의 pose-graph 최적화)

Part 5. SLAM vs Localization — 헷갈리지 말 것

SLAM Localization (AMCL)
지도 없음 → 만든다 이미 있음
푸는 것 지도 + 위치 동시 위치만
ROS2 도구 slam_toolbox nav2_amcl
시점 맵 구축 단계 (1회) 운영 중 매번

보통 워크플로: ① SLAM으로 지도 한번 만들고(.pgm+.yaml 저장) → ② 그 지도로 AMCL을 돌려 매번 위치추정. 자세한 AMCL은 [[2026-06-16-amcl]] 참고.

Part 6. ROS2 개발자 보강

  • 출력물: map_saver_cli로 map.pgm(점유격자 이미지) + map.yaml(해상도·원점) 저장 → Nav2 map_server가 로드 ([[2026-06-16-nav2-pkg]]).
  • 점유 격자(Occupancy Grid): 각 셀이 점유(100)/비점유(0)/미지(-1) 확률. nav_msgs/OccupancyGrid.
  • 필요 입력: LaserScan(/scan) + Odometry(/odom) + TF(odom→base_link). → [[2026-06-16-tf]]
  • 백엔드: 현대 SLAM은 대부분 Graph SLAM(pose-graph + loop closure 최적화). 파티클 기반(gmapping)은 구식.

한 줄 요약

SLAM은 지도도 없고 위치도 모르는 상태에서 둘을 동시에 푸는 기술이다. 이동량을 추정(오도메트리)하고 특징점을 매칭하며 지도를 키우다, 루프 클로저로 누적 드리프트를 펴서 일관된 지도를 완성한다. 지도가 완성되면 그 지도로 AMCL이 매번 위치추정을 한다 — SLAM은 지도 만들기, AMCL은 만든 지도에서 길찾기.
