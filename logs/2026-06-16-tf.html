TF 정리 — ROS2의 좌표 프레임 변환 시스템 (tf2)

TF(tf2)는 ROS2에서 여러 좌표 프레임(frame) 사이의 관계를 시간에 따라 추적·변환해주는 시스템이다. 흐름: 왜 필요한가 → 프레임 트리 → 누가 무엇을 쏘나 → static vs dynamic → 도구 → Nav2에서의 TF (좌표변환의 수학은 [[2026-06-16-coordinate-transform]], 이 노트는 ROS2 구현)

Part 0. 왜 TF가 필요한가

로봇엔 좌표계가 많다 — 지도(map), 로봇 몸체(base_link), LiDAR(laser), 카메라(camera), 바퀴… LiDAR가 "내 앞 2m에 벽" 이라고 하면, 그걸 "지도상 (5.3, 1.2) 위치의 벽" 으로 바꿔야 쓸 수 있다.

TF = "센서가 본 좌표를 다른 기준 좌표로 자동 환산해주는 시스템." 각 프레임 사이 변환을 모아 트리로 관리하고, "laser→map 변환 줘" 하면 중간 경로를 곱해서 답해준다.

Part 1. TF 트리 (Transform Tree)

프레임들은 하나의 트리로 연결된다 (사이클 없음, 각 프레임은 부모 하나):

map  →  odom  →  base_link  →  laser
                          └─→  camera
                          └─→  imu
  • 각 화살표 = 부모→자식 변환(translation + rotation)
  • 임의의 두 프레임 변환은 트리 경로를 따라 변환을 곱해서 계산
  • map↔odom↔base_link의 의미는 [[2026-06-16-odometry]] 참고 (REP-105 표준)

Part 2. 누가 무엇을 발행(broadcast)하나 — 중요!

TF 트리는 여러 노드가 각자 한 구간씩 책임진다:

변환 구간 누가 발행 성격
map → odom AMCL (또는 SLAM) 드리프트 보정분. 불연속(점프 가능)
odom → base_link 바퀴 컨트롤러 / ros2_control / robot_localization 오도메트리. 연속적
base_link → laser/camera/... robot_state_publisher (URDF 기반) 보통 고정(static)

핵심: AMCL은 map→base_link를 직접 쏘지 않는다. 이미 누군가 odom→base_link를 쏘고 있으니, AMCL은 그 위에 얹는 map→odom(보정분)만 발행한다. 그래야 트리에서 각 프레임의 부모가 하나로 유지된다. ([[2026-06-16-amcl]])

Part 3. Static vs Dynamic Transform

Static TF Dynamic TF
토픽 /tf_static /tf
언제 변하지 않는 관계 (로봇에 볼트로 고정된 센서) 시간에 따라 변함 (로봇 이동, 관절)
발행 한 번(latched) 계속
도구 static_transform_publisher TransformBroadcaster

Nav2에서 컨테이너 안 노드들이 /tf, /tf_static을 remap으로 공유하는 이유가 이것 ([[2026-06-16-nav2-pkg]]의 composition).

Part 4. transform_tolerance — 시간 동기화

TF는 시간 스탬프가 핵심이다. "0.1초 전 시점의 laser→map 변환" 처럼 과거 특정 시각의 변환을 묻는다.

  • transform_tolerance: TF가 살짝 늦게 와도 허용하는 시간 오차 (네트워크 지연 대응). 너무 작으면 "extrapolation into the future" 에러로 데이터를 버린다.
  • 흔한 버그: 노드 간 use_sim_time 불일치 → TF 시간이 안 맞아 "could not transform" 에러.

Part 5. ROS2 개발자 도구 (실무 필수)

# 프레임 트리를 PDF로 시각화
ros2 run tf2_tools view_frames

# 두 프레임 사이 실시간 변환 출력
ros2 run tf2_ros tf2_echo map base_link

# 특정 변환을 수동 발행 (테스트용)
ros2 run tf2_ros static_transform_publisher 0 0 0 0 0 0 base_link laser
  • API: C++ tf2_ros::TransformBroadcaster / TransformListener, Python tf2_ros.
  • 메시지: geometry_msgs/TransformStamped (header.frame_id=부모, child_frame_id=자식).
  • 흔한 증상 진단: RViz에서 "No transform from [X] to [map]" → 트리가 끊김(누가 그 구간을 안 쏨) 또는 시간 불일치.

한 줄 요약

TF(tf2)는 로봇의 모든 좌표 프레임을 하나의 시간 인식 트리로 묶어, "이 센서 좌표를 저 기준으로 바꿔줘"를 자동 처리하는 시스템이다. 트리는 여러 노드가 한 구간씩 나눠 발행하며(odom→base_link는 오도메트리, map→odom은 AMCL), view_frames·tf2_echo로 디버깅한다. 좌표변환의 수학적 원리는 [[2026-06-16-coordinate-transform]]에서.
